FlowPilot: منصة إدارة المشاريع والفرق والمهام

FlowPilot هو مشروع عملي سنبنيه معاً خطوة بخطوة طوال 8 أسابيع. هو ليس مجرد أمثلة منفصلة - إنه نظام متكامل ستتعلم من خلاله كل مفاهيم Laravel 12 و React و Inertia.js و Tailwind CSS في سياق تطبيق حقيقي.

ما هي فكرة FlowPilot؟

FlowPilot هي منصة سحابية (تعمل عبر المتصفح) تساعد الفرق على تنظيم مشاريعها ومهامها. الفكرة بسيطة: كل مشروع يحتوي على مهام، كل مهمة لها حالة (جديد، قيد التنفيذ، منجز)، وكل فريق يمكنه إدارة مشاريعه الخاصة.

ما يميز FlowPilot عن التطبيقات التعليمية البسيطة أنه يتعامل مع مفاهيم حقيقية موجودة في أي تطبيق ويب احترافي مثل: تسجيل الدخول، الصلاحيات، قواعد البيانات، العلاقات، رفع الملفات، البحث، التنبيهات، وسجل النشاط.

ما المشكلة التي يحلها FlowPilot؟

تخيل أن فريقاً صغيراً يعمل على مشروع برمجي. كيف يتتبعون المهام؟ كيف يعرف كل عضو ما المطلوب منه؟ كيف يعرف مدير الفريق حالة كل مهمة؟

بدون نظام إدارة مشاريع، يضيع الوقت في المراسلة والمتابعة اليدوية. FlowPilot يحل هذه المشكلة بتوفير منصة مركزية يستطيع الفريق من خلالها:

  • إنشاء مشاريع متعددة وتنظيمها.
  • إضافة مهام وتعيينها لأعضاء الفريق.
  • متابعة حالة المهام في لوحة Kanhan بصرية.
  • إرفاق ملفات بالمهام للتوثيق.
  • تسجيل كل حدث في سجل نشاط واضح.
  • استلام تنبيهات عند تغيير المهام.

كيف سنستخدم Laravel و React و Inertia و Tailwind؟

🔧 Laravel 12 (الخلفية)

Laravel هو المسؤول عن الجزء الخلفي من التطبيق (Backend). هو الذي يستقبل طلبات المستخدم من المتصفح، يتصل بقاعدة البيانات، يتحقق من صلاحيات المستخدم، ويعيد النتيجة المناسبة.

في FlowPilot، Laravel يدير: المسارات، Controllers، Models، التحقق من صحة البيانات، الاستعلامات، Policies، Middleware، Notifications، و Jobs.

⚛️ React (الواجهة)

React هو المسؤول عن واجهة المستخدم (Frontend). هو الذي يبني الصفحات التي يراها المستخدم ويتفاعل معها. كل صفحة في FlowPilot هي Component React.

في FlowPilot، React تبني: الصفحات (Pages)، الـ Layouts، الـ Components المتكررة (بطاقات المشاريع، جداول المهام، نماذج الإدخال)، ولوحة Kanban.

🔗 Inertia.js (الجسر)

Inertia هو الطبقة التي تربط Laravel بـ React. بدلاً من بناء API منفصل (REST API) والتعامل مع JSON، Inertia يسمح لـ Laravel بإرسال البيانات مباشرة إلى Components React.

في FlowPilot، Inertia يسمح لنا بكتابة التطبيق كتطبيق صفحة واحدة (SPA) دون تعقيد API منفصل.

🎨 Tailwind CSS (التصميم)

Tailwind هو إطار عمل CSS يساعدنا في تصميم الواجهات بسرعة. بدلاً من كتابة CSS مخصص لكل عنصر، نستخدم classes جاهزة مثل p-4 و text-xl و bg-indigo-600.

في FlowPilot، Tailwind يجعل كل صفحة ذات مظهر احترافي ومتجاوب مع جميع أحجام الشاشات.

ما المميزات التي سنبنيها؟

👥 إدارة المستخدمين

تسجيل، دخول، تعديل ملف شخصي، تغيير كلمة مرور.

📁 إدارة المشاريع

إنشاء، عرض، تعديل، حذف المشاريع مع وصف وتاريخ.

📋 إدارة المهام

مهام داخل المشاريع مع حالة وتحديد مسؤول وأولوية.

📊 لوحة Kanban

عرض المهام حسب الحالة مع سحب وإفلات.

🔒 صلاحيات

كل مشروع محمي بصلاحية، فقط المالك والفريق يرون المهام.

📎 مرفقات

رفع ملفات وصور للمهام مع تخزين آمن.

🔍 بحث وفلاتر

بحث عن مشاريع ومهام مع فلاتر حسب الحالة والتاريخ.

🔔 تنبيهات

إشعارات عند إسناد مهمة أو تغيير حالتها.

📝 سجل نشاط

تاريخ كل حدث في المشروع مع اسم المستخدم والتوقيت.

📈 Dashboard

لوحة تحكم بمؤشرات وإحصائيات لكل مستخدم.

كيف سيتطور المشروع أسبوعاً بعد أسبوع؟

1

الأسبوع 1: أدواتك جاهزة وأول مشروع Laravel

ننصب PHP و Composer و Node.js و Git. ننشئ أول مشروع Laravel مع React و Inertia. نرى أول صفحة في المتصفح. نحفظ المشروع في Git.

2

الأسبوع 2: المسارات والهيكل الثابت

نفهم كيف تعمل Routes و Controllers في Laravel. نبني Layout رئيسي مع Navbar و Sidebar. نضيف أول Dashboard.

3

الأسبوع 3: قاعدة البيانات والعلاقات

نصمم قاعدة بيانات FlowPilot (المشاريع، المهام، المستخدمين). ننشئ Migrations و Models. نربط البيانات بعلاقات Eloquent.

4

الأسبوع 4: المصادقة والصلاحيات

نبني نظام تسجيل الدخول. نضيف Policies لحماية المشاريع. نستخدم Middleware للصفحات الخاصة. نبني صفحة الملف الشخصي.

5

الأسبوع 5: CRUD ونماذج Inertia

نبني دورة كاملة لإنشاء وعرض وتعديل وحذف المشاريع والمهام. نضيف لوحة Kanban ورفع المرفقات. نتعامل مع Validation وأخطاء النماذج.

6

الأسبوع 6: البحث والتنبيهات

ننظم البيانات المخرجة باستخدام API Resources. نضيف بحثاً متقدماً وفلاتر. نبني نظام تنبيهات للمستخدمين. نضيف سجل نشاط لكل حدث.

7

الأسبوع 7: اختبارات وأداء وأمان

نكتب اختبارات للميزات. نحسن Components في React. نضيف Pagination و Caching للسرعة. نحمي التطبيق من الهجمات الشائعة.

8

الأسبوع 8: نشر وتسليم

نجهز المشروع للإنتاج، ننشره على خادم حقيقي، نكتب توثيقاً للمشروع، ونجهز العرض النهائي للتسليم.
