واجب: بناء TaskResource في FlowPilot

طبّق ما تعلمته عن API Resources لبناء طبقة تنظيم بيانات المهام.

📋 المهام المطلوبة بالتفصيل:

المهمة 1: إنشاء الـ Resource

شغّل أمر Artisan لإنشاء TaskResource. اكتب الأمر في سطر الأوامر داخل مجلد المشروع.

php artisan make:resource TaskResource

المهمة 2: تحديد الحقول في toArray()

افتح الملف app/Http/Resources/TaskResource.php واكتب دالة toArray() ترسل الحقول التالية:

  • id - معرف المهمة
  • title - عنوان المهمة
  • status - حالة المهمة (pending / in_progress / completed)
  • project_id - معرف المشروع التابعة له
  • assigned_user_name - اسم المستخدم المسنَد إليه (من علاقة belongsTo)
  • due_date - تاريخ التسليم بتنسيق Y-m-d

المهمة 3: إضافة حقل شرطي

أضف حقلاً اسمه priority يُرسل فقط إذا كانت المهمة ذات أولوية عالية (priority === 'high'). استخدم $this->when().

المهمة 4: استخدام الـ Resource في الـ Controller

في TaskController، عدّل دالة index() لتستخدم TaskResource::collection() بدلاً من إرسال البيانات الخام. تأكد من استيراد (import) الـ Resource في أعلى الملف.

المهمة 5: التحقق من النتيجة

افتح صفحة المهام في React وتأكد من أن البيانات تظهر بالشكل التالي في Network tab:

{ "id": 1, "title": "تصميم الصفحة الرئيسية", "status": "in_progress", "project_id": 1, "assigned_user_name": "أحمد علي", "due_date": "2025-07-01" }

📤 تسليم الواجب

  • صور (Screenshots) للـ Network tab تظهر البيانات النظيفة.
  • نسخة من ملف TaskResource.php كامل المحتوى.
  • نسخة من TaskController.php تظهر use واستخدام collection.
  • فقرة (3-5 جمل) تشرح بلسانك: لماذا نستخدم API Resources في FlowPilot؟

💡 ملاحظة للمبتدئين:

إذا واجهت خطأ "Class not found"، تأكد من إضافة use App\Http\Resources\TaskResource; في أعلى الـ Controller. إذا واجهت خطأ "Call to a member function on null"، استخدم $this->whenLoaded('relation').
