تنظيم البيانات المبتعثة: استخدام Laravel API Resources

"كما ورد في موسوعة حسوب: الموارد (Resources) هي طبقة تحويل تقع بين نماذج Eloquent وردود JSON التي تُرسل للمستخدم."

ما هو الـ API Resource؟

عندما نطلب بيانات من Laravel لتعرض في React، فإن Laravel يرسل "كل شيء" موجود في قاعدة البيانات بشكل افتراضي. لكن في المشاريع الاحترافية مثل FlowPilot، نريد التحكم الكامل فيما يظهر للمتصفح.

الـ Resource هو "مصفاة" ذكية. هو يسمح لك بإعادة تسمية الحقول (مثلاً تحويل created_at إلى date) واختيار البيانات المهمة فقط، وحجب البيانات الحساسة مثل كلمات المرور أو Tokens.

مثال لتقريب الفكرة

تخيل الـ Resource كـ "قائمة الطعام" في مطعم فاخر.

المطبخ (قاعدة البيانات) مليء بالمكونات الخام، النفايات، والأدوات. لكن الزبون (المتصفح) لا يرى كل ذلك؛ هو فقط يستلم "طبقاً منسقاً" (JSON Resource) يحتوي فقط على ما طلبه وبشكل جميل ومرتب.

التطبيق العملي

1

إنشاء مورد للمشاريع

اكتب الأمر التالي في Terminal:

php artisan make:resource ProjectResource

افتح الملف في app/Http/Resources/ProjectResource.php وحدد الحقول التي تريدها:

public function toArray(Request $request): array

{

  return [

    'id' => $this->id,

    'title' => $this->name,

    'formatted_date' => $this->created_at->diffForHumans(),

    'owner' => $this->user->name,

  ];

}
