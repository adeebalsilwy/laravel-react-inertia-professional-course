API Resources: كيف نرسل البيانات للواجهة بذكاء؟

"لا ترسل كل شيء؛ أرسل فقط ما تحتاجه الواجهة لتبدو منظمة وسريعة."

ما هو الـ API Resource؟

قاعدة البيانات تحتوي على معلومات كثيرة قد لا نحتاجها في الواجهة (مثل updated_at أو password). الـ API Resource هو طبقة تنظيمية تسمح لنا باختيار "ماذا" نرسل و "كيف" يظهر شكله في React.

في FlowPilot، سنستخدمه لنتأكد من أن بيانات المشاريع والمهام تصل للواجهة بتنسيق موحد وجميل، مما يسهل عملنا في React.

لماذا ننظف البيانات؟

  • الأمان: منع تسريب بيانات حساسة بالخطأ.
  • الأداء: تقليل حجم البيانات المرسلة عبر الإنترنت.
  • التوحيد: جعل أسماء الحقول واضحة (مثلاً تغيير is_active إلى status).

مثال لتقريب الفكرة (فلتر المياه)

تخيل قاعدة البيانات هي "خزان مياه" كبير يحتوي على شوائب وأملاح. الـ API Resource هو "الفلتر".

أنت لا تريد شرب الماء مباشرة من الخزان؛ بل تمرره عبر الفلتر ليخرج لك ماءً نقياً وصالحاً للشرب (الاستخدام في الواجهة).

التطبيق العملي

الخطوة 1: إنشاء Resource للمشروع

php artisan make:resource ProjectResource

الخطوة 2: تحديد البيانات المرسلة

داخل ملف app/Http/Resources/ProjectResource.php:

return [

  'id' => $this->id,

  'name' => $this->name,

  'tasks_count' => $this->tasks()->count(),

];
