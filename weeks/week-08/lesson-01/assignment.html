واجب الدرس 29: تجهيز الـ Build النهائي وملفات البيئة

"حول مشروع FlowPilot من نسخة تطوير إلى نسخة إنتاج جاهزة للعالم الحقيقي."

هدف الواجب

الهدف من هذا الواجب هو تطبيق كل ما تعلمته في الدرس على مشروع FlowPilot الحقيقي. بعد الانتهاء، يجب أن تكون قادراً على شرح الفرق بين بيئة التطوير والإنتاج، وأهمية كل متغير في ملف .env.

المطلوب تنفيذه (بالترتيب)

الخطوة 1: بناء ملفات الإنتاج

افتح Terminal في مجلد مشروع FlowPilot واكتب:

npm run build

تأكد من انتهاء الأمر دون أخطاء.

الخطوة 2: فحص الناتج

افتح مجلد المشروع وتأكد من وجود:

public/build/assets/
public/build/manifest.json

manifest.json هو ملف يخبر Laravel بأسماء الملفات المضغوطة الجديدة.

الخطوة 3: تعديل ملف .env

افتح ملف .env في جذر المشروع وغير القيم التالية:

APP_ENV=production
APP_DEBUG=false
APP_URL=http://localhost:8000

إذا كان عندك رابط مختلف استخدمه بدلاً من localhost.

الخطوة 4: تشغيل أوامر التحسين

نفذ الأوامر التالية بالترتيب:

php artisan optimize
php artisan config:cache
php artisan route:cache

لاحظ ظهور رسالة تأكيد بعد كل أمر.

الخطوة 5: اختبار النتيجة

شغل السيرفر المحلي:

php artisan serve

افتح المتصفح على الرابط http://localhost:8000. تأكد من أن الموقع يعمل بشكل طبيعي. حاول الدخول إلى رابط غير موجود (مثل /test404) - يجب أن تظهر صفحة خطأ بسيطة بدلاً من التفاصيل التقنية.

أسئلة للإجابة (سلم مع الواجب)

  1. ما الفرق بين npm run dev و npm run build؟ اشرح بجملتين.
  2. ماذا يحدث إذا تركت APP_DEBUG=true في موقع الإنتاج؟
  3. ما هو دور ملف manifest.json الذي يظهر بعد build؟
  4. لماذا نستخدم php artisan config:cache قبل نشر الموقع؟
  5. كم حجماً أصبحت ملفات JavaScript بعد build مقارنة بما كانت عليه في التطوير؟
