بناء القالب الرئيسي (Layout): كيف لا نكرر كود الواجهة؟

"المبرمج الذكي هو من يكتب الكود مرة واحدة ويستخدمه في كل مكان."

لماذا نحتاج الـ Layout؟

تخيل أن تطبيق FlowPilot يحتوي على 50 صفحة. في كل صفحة نحتاج لشريط تنقل (Navbar) في الأعلى وقائمة جانبية (Sidebar). هل يعقل أن ننسخ كود الـ Navbar في 50 ملفاً؟

بالطبع لا. إذا أردنا تغيير لون الـ Navbar لاحقاً، سيتعين علينا تعديل 50 ملفاً! الحل هو الـ Layout؛ وهو قالب ثابت نضع فيه العناصر المشتركة، ثم نضع محتوى كل صفحة "بداخله".

قاعدة DRY البرمجية

في البرمجة هناك قاعدة ذهبية تسمى DRY وهي اختصار لـ Don't Repeat Yourself (لا تكرر نفسك). الـ Layout هو تطبيق عملي لهذه القاعدة في تصميم الواجهات.

<Layout>
  {/* محتوى الصفحة يتغير هنا */}
  {children}
</Layout>

مثال من الواقع (إطار الصور)

تخيل الـ Layout كـ "إطار خشبي" جميل معلق على الحائط. الإطار ثابت لا يتغير مكانه ولا شكله.

أما محتوى الصفحة فهو "الصورة" التي تضعها داخل هذا الإطار. يمكنك تغيير الصورة في أي وقت (تبديل الصفحات)، لكن الإطار يظل يحميها ويزينها بشكل ثابت.

التطبيق العملي خطوة بخطوة

الخطوة 1: إنشاء ملف Layout.jsx

المكان: مجلد resources/js/Layouts/AppLayout.jsx

export default function AppLayout({ children }) {

  return (

    <div class="min-h-screen bg-gray-100">

      <nav class="bg-white shadow">...شريط التنقل...</nav>

      <main>{children}</main>

    </div>

  );

}

الخطوة 2: استخدام القالب في صفحاتنا

افتح ملف Dashboard.jsx وقم بتغليفه بالقالب الجديد.

import AppLayout from '../Layouts/AppLayout';

Dashboard.layout = page => <AppLayout children={page} />
