📄 واجب الدرس 6: بناء الهيكل الموحد (Layout)

"القالب الجيد يوفر عليك ساعات من التكرار. ابنِ قالب FlowPilot اليوم."

🎯 الهدف من الواجب

بعد إتمام هذا الواجب، ستكون قادراً على:

  • إنشاء Layout رئيسي لتطبيق FlowPilot.
  • تصميم Navbar و Sidebar كاملين.
  • ربط Layout بصفحات متعددة دون تكرار الكود.
  • إضافة تمييز للرابط النشط.
  • جعل التصميم متجاوباً (Responsive).

📋 خطوات الواجب

1

إنشاء مجلد Layouts وملف AppLayout.jsx

أنشئ مجلد resources/js/Layouts/ وفيه ملف AppLayout.jsx.

import { useState } from 'react'; import { Link } from '@inertiajs/react'; export default function AppLayout({ children }) {   const [sidebarOpen, setSidebarOpen] = useState(true);   return (     <div className="min-h-screen bg-gray-100 flex">       <aside className="w-64 bg-white shadow-lg">         {/* Sidebar content here */}       </aside>       <main className="flex-1 p-6">{children}</main>     </div>   ); }
2

تصميم شريط التنقل العلوي (Navbar)

أضف داخل AppLayout Header في الأعلى يحتوي على:

  • شعار FlowPilot على اليسار (لأن التوجيه RTL).
  • زر القائمة (Hamburger) للموبايل.
  • اسم المستخدم (يمكنك استخدام نص وهمي).
3

تصميم القائمة الجانبية (Sidebar)

أضف 4 روابط على الأقل في Sidebar:

  • 🏠 Dashboard ← /dashboard
  • 📁 المشاريع ← /projects
  • ✅ المهام ← /tasks
  • 👥 الفريق ← /team

استخدم مكون <Link> من Inertia بدلاً من <a>.

4

إضافة تمييز الرابط النشط

استخدم route().current() لتغيير لون الرابط الحالي. الرابط النشط يجب أن يكون له خلفية bg-indigo-100 ولون text-indigo-700.

5

ربط Layout بصفحة Dashboard

افتح Dashboard.jsx وأضف السطر التالي بعد تعريف الدالة:

Dashboard.layout = page => <AppLayout children={page} />;
6

جعل التصميم متجاوباً (Responsive)

أضف كلاسات Tailwind التالية:

  • على Sidebar: hidden md:block (يختفي في الموبايل)
  • على زر القائمة: md:hidden (يظهر فقط في الموبايل)
  • استخدم useState للتحكم في إظهار/إخفاء القائمة.

✅ معايير القبول

  • الـ Layout يعمل مع Dashboard ويظهر فيها Navbar + Sidebar.
  • الرابط النشط يظهر بلون مختلف عن باقي الروابط.
  • على شاشة الموبايل، تختفي القائمة ويظهر زر لفتحها.
  • لا يوجد أخطاء في Console المتصفح.
  • استخدام <Link> من Inertia (وليس <a>).

⭐ تحديات إضافية

  • قسّم AppLayout إلى مكونات منفصلة: Navbar.jsx و Sidebar.jsx و Footer.jsx.
  • أضف Footer في أسفل كل صفحة بعبارة "© 2026 FlowPilot".
  • أضف صورة مستخدم وهمية (Avatar) في Navbar باستخدام <img>.
  • استخدم usePage() من Inertia لعرض اسم المستخدم الحقيقي (إذا كان مسجلاً).

📬 طريقة التسليم

  1. لقطة شاشة للصفحة في وضع سطح المكتب (Desktop) تظهر الـ Layout كاملاً.
  2. لقطة شاشة للصفحة في وضع الموبايل (Mobile View) تظهر زر القائمة.
  3. نسخ محتوى الملفات: AppLayout.jsx، Dashboard.jsx.
  4. أرسل كل ما سبق إلى المدرب للمراجعة.
