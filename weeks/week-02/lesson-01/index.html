فهم المسارات (Routing) وكيف يرسل Laravel البيانات لـ React

"المسار هو الباب الذي يدخل منه المستخدم لتطبيقك. لنتعلم كيف نبني أبوابنا الذكية."

لماذا نحتاج الـ Routing؟

عندما تكتب facebook.com/profile، كيف يعرف الفيسبوك أنه يجب عرض صفحتك الشخصية؟ هذا هو دور الـ Routing. هو نظام توجيه يربط "الرابط" الذي يطلبه المستخدم بـ "الكود" الذي يجب أن يعمل.

في مشروعنا FlowPilot، سنحتاج لمسارات كثيرة مثل: مسار لعرض المهام، ومسار لإنشاء مشروع جديد، ومسار لتعديل البيانات.

كيف يعمل Inertia Response؟

في Laravel التقليدي، كنا نعيد ملف Blade. لكننا الآن نستخدم React. هنا يأتي دور Inertia::render(). هي دالة تخبر Laravel: "توقف، لا ترسل HTML عادياً، بل أرسل البيانات لصفحة React الفلانية لتقوم هي بالعرض".

Route::get('/dashboard', function() {
  return Inertia::render('Dashboard');
});

مثال لتقريب الفكرة

تخيل التطبيق كـ "مجمع تجاري".
- الـ Route هو اللوحات الإرشادية التي تخبرك: "للوصول لقسم الملابس اذهب يميناً".
- المتصفح هو الزائر.
- Inertia هو الموظف الذي يفتح لك باب القسم المخصص ويعرض لك البضائع (البيانات) بداخله.

التطبيق العملي

1

إنشاء مسار Dashboard

افتح ملف routes/web.php وأضف الكود التالي:

use Inertia\Inertia;

Route::get('/dashboard', function () {

  return Inertia::render('Dashboard', [

    'userName' => 'أحمد'

  ]);

});

هنا قمنا بربط الرابط /dashboard بصفحة React تسمى Dashboard.jsx وأرسلنا لها بيانات (اسم المستخدم).
