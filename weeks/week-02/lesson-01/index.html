فهم المسارات (Routing) وكيف يربط Laravel الطلبات بصفحات React

"المسار هو الباب الذي يدخل منه المستخدم إلى تطبيقك. لنتعلم كيف نصنع أبواباً ذكية لمشروع FlowPilot."

🔍 لماذا هذا الدرس مهم؟

تخيل أن تطبيق FlowPilot عبارة عن فندق ضخم. كل غرفة في هذا الفندق هي صفحة من صفحات التطبيق. بدون ممرات وأرقام غرف (أي بدون مسارات)، لن يعرف الزائر كيف يصل إلى غرفته!

في عالم البرمجة، الـ Route (ويُلفظ "روت" ويعني "مسار") هو الرابط الذي يكتبه المستخدم في المتصفح. مثلاً: flowpilot.com/projects أو flowpilot.com/dashboard. مهمة الـ Route هي أن يأخذ هذا الرابط ويقرره: "أي كود يجب أن يعمل؟ وأي صفحة يجب أن تظهر؟"

بدون فهم المسارات، لا يمكنك بناء أي تطبيق ويب متعدد الصفحات. هذا الدرس هو أساس كل ما سنبنيه في FlowPilot.

📚 الشرح النظري المفصل

ما هو الـ Route بالضبط؟

الـ Route هو عبارة عن "قاعدة" في ملف routes/web.php تخبر Laravel: "إذا جاء مستخدم إلى هذا الرابط، افعل كذا".

مثال توضيحي: عندما يكتب المستخدم https://flowpilot.com/projects في المتصفح، يبحث Laravel في ملف web.php عن Route يناسب هذا الرابط. إذا وجده، ينفذ الكود المرتبط به.

أنواع المسارات (HTTP Methods)

هناك عدة أنواع من المسارات، لكن الأكثر شيوعاً:

  • Route::get() - يستخدم عندما يريد المستخدم رؤية صفحة (مثلاً: عرض صفحة المشاريع).
  • Route::post() - يستخدم عندما يريد المستخدم إرسال بيانات (مثلاً: تسجيل الدخول أو إنشاء مشروع جديد).
  • Route::put() / patch() - لتعديل بيانات موجودة.
  • Route::delete() - لحذف بيانات.

ما هو الـ Controller؟

الـ Controller (ويعني "وحدة التحكم") هو ملف منفصل ينظم الكود الخاص بصفحة معينة. بدلاً من كتابة كل الكود داخل ملف web.php (مما يجعله فوضوياً)، نكتب الكود في Controller ونستخدم سطراً واحداً فقط في ملف المسارات.

مثال: بدلاً من:

Route::get('/projects', function () {   // 50 سطر من الكود هنا!   return Inertia::render('Projects/Index'); });

نكتب:

Route::get('/projects', [ProjectController::class, 'index'])   ->name('projects.index');

وهذا أنظف بكثير! الكود الفعلي سيكون في ProjectController.php.

ما هو Inertia::render()؟

في تطبيقات Laravel التقليدية، كنا نستخدم return view('welcome') لعرض صفحة Blade (HTML عادي). أما الآن مع Inertia، نستخدم:

return Inertia::render('Projects/Index', [   'projects' => $projects ]);

ماذا يفعل هذا السطر؟

  • Inertia::render() تخبر Laravel: "لا ترسل HTML عادياً، بل أرسل البيانات إلى React".
  • المعامل الأول 'Projects/Index' هو اسم ملف React الذي سيتم عرضه (الموجود في resources/js/Pages/Projects/Index.jsx).
  • المعامل الثاني ['projects' => $projects] هو البيانات التي نريد إرسالها إلى صفحة React.

ما هي Named Routes (المسارات المسماة)؟

بدلاً من استخدام الرابط مباشرة في كل مكان (مثل /projects)، نعطي كل Route اسماً. هذا أفضل لأنه إذا تغير الرابط يوماً ما، نغير في مكان واحد فقط!

Route::get('/projects', ...)->name('projects.index');

في React، نستخدم دالة route() التي يوفرها Inertia لاستخدام الاسم:

<Link href={route('projects.index')}>المشاريع</Link>

🌍 مثال من الحياة اليومية

تشبيه "المطعم الذكي"

تخيل أنك تدخل مطعماً. ما الذي يحدث؟

  • المتصفح (Browser): أنت كزائر تدخل المطعم.
  • الرابط (URL): هو عنوان المطعم (مثلاً: مطعم السلام في شارع الملك).
  • الـ Route: هو "النادل" الذي يستقبلك ويسألك: "طاولة للأكل أم طلب سفري؟". هو يقرر أين يوجهك بناءً على طلبك.
  • الـ Controller: هو " chef الطاهي" في المطبخ. عندما يطلب النادل طبقاً معيناً، chef هو من يحضره.
  • Inertia::render(): هو "صينية التقديم" التي يحمل عليها النادل الطبق (البيانات) ليقدمها لك بشكل جميل.
  • صفحة React: هي "الطبق" نفسه بعد تقديمه وتزيينه.

المهم أن تفهم: كل طلب يمر بنفس الرحلة: زائر ← نادل ← طاهٍ ← تقديم. وفي البرمجة: متصفح ← Route ← Controller ← Inertia ← صفحة React.

🚀 كيف يخدم هذا الدرس مشروع FlowPilot؟

في مشروع FlowPilot، سنحتاج إلى عدة صفحات:

  • /dashboard ← لوحة التحكم الرئيسية.
  • /projects ← عرض كل المشاريع.
  • /projects/create ← إنشاء مشروع جديد.
  • /tasks ← عرض المهام.
  • /team ← صفحة الفريق.

كل صفحة من هذه الصفحات تحتاج إلى Route في ملف web.php، و Controller لمعالجة البيانات، وصفحة React لعرضها. هذا الدرس يؤسس لهذه البنية التحتية.

⚡ التطبيق العملي خطوة بخطوة

الخطوة 1: إنشاء Controller لمشاريع FlowPilot

المكان: نافذة الطرفية (Terminal) في مشروع Laravel

نستخدم أمر Artisan لإنشاء Controller جديد:

php artisan make:controller ProjectController

ماذا حدث؟ تم إنشاء ملف جديد في المسار app/Http/Controllers/ProjectController.php. هذا الملف سيكون مسؤولاً عن كل العمليات المتعلقة بالمشاريع.

ملاحظة للمبتدئين: php artisan هو اسم أداة Laravel السحرية. أي أمر يبدأ بـ php artisan هو أمر يساعدك في إنشاء ملفات جاهزة بدلاً من كتابتها يدوياً.

الخطوة 2: كتابة دالة index في Controller

المكان: app/Http/Controllers/ProjectController.php

<?php namespace App\Http\Controllers; use Inertia\Inertia; class ProjectController extends Controller {   public function index()   {     $projects = [       ['id' => 1, 'title' => 'موقع الشركة', 'status' => 'نشط'],       ['id' => 2, 'title' => 'تطبيق الجوال', 'status' => 'قيد التطوير'],       ['id' => 3, 'title' => 'لوحة التحكم', 'status' => 'مكتمل'],     ];     return Inertia::render('Projects/Index', [       'projects' => $projects     ]);   } }

ملاحظة مهمة: في هذه المرحلة نستخدم بيانات "وهمية" (مصفوفة مكتوبة يدوياً). لاحقاً في الأسبوع 3 سنستبدلها ببيانات حقيقية من قاعدة البيانات.

الخطوة 3: تعريف المسار في routes/web.php

المكان: routes/web.php

<?php use Illuminate\Support\Facades\Route; use App\Http\Controllers\ProjectController; Route::get('/', function () {   return Inertia::render('Welcome'); }); Route::get('/projects', [ProjectController::class, 'index'])   ->name('projects.index');

شرح السطر الجديد:

  • Route::get('/projects', ...) يعني: عندما يزور المستخدم الرابط /projects...
  • [ProjectController::class, 'index'] يعني: اذهب إلى ملف ProjectController ونفذ الدالة التي اسمها index.
  • ->name('projects.index') يعني: أعط هذا المسار اسماً هو projects.index حتى نستخدمه لاحقاً في الروابط.

الخطوة 4: إنشاء صفحة React لعرض المشاريع

المكان: resources/js/Pages/Projects/Index.jsx

import { Link } from '@inertiajs/react'; export default function ProjectsIndex({ projects }) {   return (     <div className="p-8">       <h1 className="text-3xl font-black mb-6">المشاريع</h1>       <div className="grid gap-4">         {projects.map(project => (           <div key={project.id} className="p-4 border rounded-xl shadow-sm">             <h2 className="font-bold text-xl">{project.title}</h2>             <p>الحالة: {project.status}</p>           </div>         ))}       </div>     </div>   ); }

الخطوة 5: إضافة رابط لصفحة المشاريع في الواجهة

المكان: أي صفحة React أخرى تريد إضافة رابط منها

import { Link } from '@inertiajs/react'; // داخل JSX: <Link   href={route('projects.index')}   className="px-4 py-2 bg-indigo-600 text-white rounded-xl" >   عرض المشاريع </Link>

ملاحظة: دالة route() متوفرة تلقائياً في صفحات React عند استخدام Inertia. هي تترجم اسم المسار (مثل projects.index) إلى الرابط الفعلي (مثل /projects). هذا يعني أنه إذا غيرنا الرابط في web.php لاحقاً، كل الروابط في React ستتحدّث تلقائياً!

الخطوة 6: اختبار المسار

المكان: المتصفح

php artisan serve

افتح المتصفح واذهب إلى: http://localhost:8000/projects

يجب أن ترى صفحة المشاريع مع عناوين المشاريع الوهمية الثلاثة التي أنشأناها.

📝 شرح الكود سطراً سطراً

ملف routes/web.php

1 <?php
2
3 use Illuminate\Support\Facades\Route;
4 use App\Http\Controllers\ProjectController;
5
6 Route::get('/projects', [ProjectController::class, 'index'])
7   ->name('projects.index');

السطر 3: use Illuminate\Support\Facades\Route;

هذا السطر يستورد (import) مكتبة Route من Laravel. تخيل أنك تستعير صندوق أدوات من جارك - هذا السطر يقول "أنا بحاجة لصندوق أدوات Route".

السطر 4: use App\Http\Controllers\ProjectController;

هنا نستورد ملف ProjectController الذي أنشأناه. هذا يخبر Laravel بمكان ملف الـ Controller.

السطر 6-7: تعريف المسار

Route::get('/projects', ...): هذا هو تعريف المسار. get تعني أن هذا المسار يستجيب لطلبات العرض فقط.

[ProjectController::class, 'index']: هذا يشير إلى أن الدالة index في ProjectController هي المسؤولة عن معالجة هذا المسار.

->name('projects.index'): نعطي المسار اسماً لاستخدامه في الروابط لاحقاً.

ملف ProjectController.php (دالة index)

1 public function index()
2 {
3   $projects = [ /* المصفوفة */ ];
4   return Inertia::render('Projects/Index', [
5     'projects' => $projects
6   ]);
7 }

السطر 3: $projects = [...]

ننشئ مصفوفة (array) تحتوي على بيانات المشاريع. المصفوفة هي مجموعة من العناصر، تماماً مثل قائمة التسوق. كل مشروع له id، title، و status.

السطر 4-6: return Inertia::render(...)

Inertia::render('Projects/Index', ...) الجزء الأول هو مسار ملف React (بدون .jsx). الجزء الثاني هو البيانات التي نرسلها كمصفوفة (key => value).

'projects' => $projects: المفتاح 'projects' سيصبح هو اسم Prop الذي سنستقبله في React.

✅ النتيجة المتوقعة

بعد تنفيذ الخطوات، يجب أن ترى في المتصفح:

  • شريط عنوان المتصفح يظهر: http://localhost:8000/projects
  • صفحة بيضاء تحتوي على عنوان "المشاريع" في الأعلى.
  • تحت العنوان، 3 بطاقات (cards) تعرض المشاريع:
    • موقع الشركة - الحالة: نشط
    • تطبيق الجوال - الحالة: قيد التطوير
    • لوحة التحكم - الحالة: مكتمل
  • البطاقات مرتبة في عمود واحد (يمكن تحسينها لاحقاً باستخدام Grid).

⚠️ الأخطاء الشائعة

🚫 الخطأ 1: صفحة 404 (الصفحة غير موجودة)

السبب: لم يتم تعريف المسار في web.php، أو أن الرابط الذي كتبته غير مطابق.

الحل: تأكد من أن الرابط في المتصفح يطابق بالضبط ما كتبته في Route::get(). مثلاً إذا كتبت /projects في الكود، فتأكد أنك لا تتصفح /project (بدون حرف s).

🚫 الخطأ 2: Class "ProjectController" not found

السبب: نسيت استيراد ProjectController في أعلى ملف web.php.

الحل: أضف السطر use App\Http\Controllers\ProjectController; في بداية الملف.

🚫 الخطأ 3: Target class [ProjectController] does not exist

السبب: الملف لم يُنشأ بعد، أو اسم الملف خطأ.

الحل: شغّل الأمر php artisan make:controller ProjectController وتأكد من ظهور الملف في المجلد الصحيح.

🚫 الخطأ 4: صفحة بيضاء فارغة (White Screen)

السبب: عادةً خطأ في بناء JavaScript (Syntax error) في ملف React.

الحل: افتح وحدة التحكم في المتصفح (F12 ثم تبويب Console) وابحث عن الخطأ الأحمر. الأكثر شيوعاً هو نسيان قوس إغلاق ) أو }.

🚫 الخطأ 5: البيانات لا تظهر في الصفحة (undefined)

السبب: اسم Prop في React لا يطابق المفتاح الذي أرسلته من Controller.

الحل: تأكد من أن المفتاح في Inertia::render('Page', ['projects' => $data]) يطابق اسم المتغير في export default function Page({ projects }).

📋 ملخص الدرس

المفاهيم الرئيسية

  • 🔹 Route: هو الرابط الذي يوجه الطلب إلى الكود المناسب.
  • 🔹 Controller: ملف ينظم كود معالجة الطلبات.
  • 🔹 Inertia::render(): دالة ترسل البيانات إلى صفحات React.
  • 🔹 Named Routes: أسماء للمسارات لتسهيل استخدامها في الروابط.
  • 🔹 route() helper: دالة تحول اسم المسار إلى رابط فعلي.

المصطلحات المهمة

  • 🔸 HTTP GET: طلب لعرض البيانات (تستخدمه عندما تتصفح صفحة).
  • 🔸 HTTP POST: طلب لإرسال بيانات (تستخدمه عند تسجيل الدخول).
  • 🔸 Artisan: أداة Laravel لإنشاء الملفات تلقائياً.
  • 🔸 Props: بيانات تُمرر من Controller إلى صفحة React.
  • 🔸 JSX: لغة تشبه HTML ولكن داخل JavaScript.
