مكونات React (Components): بناء تطبيقك كقطع تركيب

"لا تبنِ صفحة كاملة دفعة واحدة، بل ابنِ قطعاً صغيرة واجمعها لتصنع تحفة برمجية."

ما هو الـ Component؟

المكون (Component) هو عبارة عن "قطعة" صغيرة من واجهة المستخدم لها وظيفة وشكل محدد. تخيل الزر، أو حقل الإدخال، أو بطاقة المشروع (Project Card). كل واحد من هذه العناصر هو مكون مستقل.

في FlowPilot، بدلاً من كتابة كود جدول المهام في كل مكان، سنصنع مكوناً اسمه TaskTable ونستخدمه في أي صفحة نحتاجها بكلمة واحدة فقط!

لماذا نقسم الواجهة إلى أجزاء؟

هناك ثلاثة أسباب رئيسية تجعلنا نستخدم المكونات:

  • إعادة الاستخدام: تصمم الزر مرة واحدة وتستخدمه في مئات الصفحات.
  • سهولة الصيانة: إذا أردت تغيير لون كل الأزرار، تعدل ملفاً واحداً فقط.
  • تنظيم الكود: بدلاً من ملف واحد يحتوي على 1000 سطر، سيكون لديك 10 ملفات صغيرة، كل واحد منها سهل الفهم.

مثال من حياتنا (السيارة)

تخيل السيارة كأنها تطبيق ويب. هل تُصنع السيارة كقطعة حديد واحدة صلبة؟ مستحيل!

السيارة هي مجموعة من المكونات: المحرك، العجلات، المقاعد، المصابيح. كل قطعة تُصنع وتُختبر وحدها، ثم تُجمع معاً. إذا تعطل المصباح، لا نغير السيارة بالكامل، بل نغير "مكون المصباح" فقط!

التطبيق العملي خطوة بخطوة

الخطوة 1: إنشاء مكون البطاقة (Card)

المكان: resources/js/Components/ProjectCard.jsx

export default function ProjectCard({ title, description }) {

  return (

    <div className="p-4 border rounded shadow">

      <h4 className="font-bold">{title}</h4>

      <p>{description}</p>

    </div>

  );

}

الخطوة 2: استدعاء المكون في الصفحة

افتح ملف Dashboard.jsx واستخدم المكون الجديد.

import ProjectCard from '@/Components/ProjectCard';

// داخل الـ return

<ProjectCard title="مشروع الموقع" description="تصميم الواجهة" />
