Migrations و Seeders و Factories للمبتدئين

"Migrations هي المخطط المعماري لقاعدة بياناتك. Seeders هي أول الأثاث الذي تضعه فيه."

مدة القراءة: 60 دقيقة مستوى: مبتدئ أدوات: Artisan, Migrations, Seeders, Factories

مقدمة: ما هي الـ Migration (الهجرة)؟

في الدرس السابق، تعلمنا كيف نخطط لقاعدة البيانات على الورق. الآن حان وقت التنفيذ الفعلي. في Laravel، لا نذهب إلى MySQL ونكتب أوامر SQL يدوياً. بدلاً من ذلك، نستخدم Migrations.

ما هي الـ Migration؟ هي ملف PHP يصف تغييراً نريد تطبيقه على قاعدة البيانات. مثلاً: "إنشاء جدول المشاريع" أو "إضافة عمود الحالة إلى جدول المهام". كل ملف Migration يشبه أمر بناء في سجل التغييرات.

لماذا لا نكتب SQL مباشرة؟ لأننا نريد تتبع التغييرات مع الوقت. مع Migrations، يمكنك:

  • إنشاء الجداول بأمر واحد
  • التراجع عن التغييرات (Rollback) بسهولة
  • مشاركة هيكل قاعدة البيانات مع فريق العمل
  • تطبيق نفس التغييرات على عدة بيئات (تطوير، اختبار، إنتاج)

📌 تشبيه بسيط:

تخيل أنك تبني بيتاً. Migration هو المخطط الهندسي الذي يخبر العمال: "هنا الجدار، هنا الباب، هنا النافذة". أنت لا تبني بالحدس - أنت تتبع مخططاً. Laravel يقرأ المخطط (ملف Migration) وينفذه على قاعدة البيانات بدقة.

إنشاء وتشغيل Migrations باستخدام Artisan

Artisan هو سطر الأوامر الخاص بـ Laravel. نستخدمه لإنشاء ملفات Migration بأمر بسيط:

# نكتب هذا الأمر في Terminal (نافذة الأوامر) داخل مجلد المشروع
$ php artisan make:migration create_projects_table

بعد تنفيذ الأمر، سيتم إنشاء ملف جديد في المسار database/migrations/. اسم الملف يبدأ بتاريخ ووقت الإنشاء، مثلاً: 2025_06_19_120000_create_projects_table.php.

⚠️ قاعدة تسمية Migrations:

Laravel يستخدم اتفاقية تسمية: create_[table_name]_table لإنشاء جدول جديد، و add_[column]_to_[table]_table لإضافة عمود لجدول موجود. اتبع هذه الاتفاقية دائماً.

تشغيل الـ Migrations:

$ php artisan migrate

هذا الأمر ينفذ جميع Migrations التي لم تُنفذ بعد. سيقرأ Laravel كل ملف Migration وينفذ up() method (الذي سنشرحه لاحقاً).

أوامر مهمة مع Migrations:

الأمر الوظيفة
php artisan migrateتنفيذ جميع Migrations الجديدة
php artisan migrate:rollbackالتراجع عن آخر batch من Migrations
php artisan migrate:refreshإعادة تنفيذ جميع Migrations (rollback + migrate)
php artisan migrate:freshحذف جميع الجداول وإعادة إنشائها
php artisan migrate:statusعرض حالة كل Migration (نفذ أم لا)

فهم up() و down() وأنواع الأعمدة

كل ملف Migration يحتوي على طريقتين (methods) رئيسيتين:

⬆️ up()

هذه الطريقة تُنفذ عند تشغيل php artisan migrate. هنا نضع الكود الذي يُنشئ الجداول أو يُعدّلها. مثلاً: Schema::create('projects', function (Blueprint $table) { ... }).

"up" تعني "الارتقاء" - ننتقل بالنسخة الحالية إلى إصدار أحدث من قاعدة البيانات.

⬇️ down()

هذه الطريقة تُنفذ عند تشغيل php artisan migrate:rollback. وظيفتها عكس ما فعلته up(). إذا أنشأت جدولاً، هنا تحذفه: Schema::dropIfExists('projects').

"down" تعني "الهبوط" - نعود للإصدار السابق. مهم جداً لتصحيح الأخطاء.

أنواع الأعمدة (Column Types) في Laravel

داخل up()، نستخدم Schema::create مع Blueprint $table. الكائن $table يوفر دوالاً لكل نوع عمود:

الدالة نوع العمود في SQL الاستخدام
$table->id()bigIncrementsالمفتاح الأساسي (PK)
$table->string('name', 255)VARCHAR(255)نص قصير (اسم، عنوان)
$table->text('description')TEXTنص طويل (وصف، محتوى)
$table->foreignId('user_id')BIGINT UNSIGNEDمفتاح أجنبي (FK)
$table->boolean('is_completed')TINYINT(1)قيمة منطقية (صح/خطأ)
$table->enum('status', ['a','b'])ENUMقيمة من قائمة محددة
$table->date('deadline')DATEتاريخ (YYYY-MM-DD)
$table->dateTime('due_date')DATETIMEتاريخ ووقت كامل
$table->timestamps()TIMESTAMPيضيف created_at و updated_at
$table->softDeletes()TIMESTAMPيضيف deleted_at للحذف الناعم
$table->nullableTimestamps()TIMESTAMP NULLtimestamps لكن اختيارية

تعديلات الأعمدة (Modifiers):

يمكننا تعديل خصائص العمود باستخدام دوال إضافية:

$table->string('email')->unique(); // قيمة فريدة (لا تتكرر) $table->string('nickname')->nullable(); // يمكن أن يكون NULL (اختياري) $table->string('color')->default('blue'); // قيمة افتراضية $table->foreignId('user_id')->constrained(); // ينشئ FK تلقائياً $table->foreignId('project_id')->constrained()->cascadeOnDelete(); // يحذف المهام تلقائياً عند حذف المشروع

تطبيق FlowPilot: إنشاء جدول Projects مع مفتاح أجنبي

الآن سنطبق ما تعلمناه. سننشئ Migration لجدول projects في مشروع FlowPilot.

الخطوة 1: إنشاء ملف Migration

$ php artisan make:migration create_projects_table

سيتم إنشاء ملف في database/migrations/2025_06_19_XXXXXX_create_projects_table.php.

الخطوة 2: كتابة الكود داخل الـ Migration

<?php use Illuminate\Database\Migrations\Migration; use Illuminate\Database\Schema\Blueprint; use Illuminate\Support\Facades\Schema; return new class extends Migration { /** * Run the migrations. */ public function up(): void { Schema::create('projects', function (Blueprint $table) { $table->id(); // المفتاح الأساسي (PK) $table->foreignId('user_id') // المفتاح الأجنبي (FK) ->constrained() // يشير تلقائياً لـ users.id ->cascadeOnDelete(); // عند حذف المستخدم، تُحذف مشاريعه $table->string('name'); // اسم المشروع (VARCHAR) $table->text('description')->nullable(); // وصف المشروع (اختياري) $table->string('status') // حالة المشروع ->default('pending'); // القيمة الافتراضية: pending $table->date('deadline')->nullable(); // تاريخ التسليم (اختياري) $table->timestamps(); // created_at + updated_at $table->softDeletes(); // deleted_at للحذف الناعم }); } /** * Reverse the migrations. */ public function down(): void { Schema::dropIfExists('projects'); // حذف الجدول عند التراجع } };

الخطوة 3: تشغيل الـ Migration

$ php artisan migrate

بعد تنفيذ الأمر، ستظهر رسالة نجاح: Migrated: 2025_06_19_XXXXXX_create_projects_table. اذهب إلى قاعدة البيانات - ستجد جدول projects قد أنشئ بجميع الأعمدة.

Seeders: زراعة البيانات الأولية

بعد إنشاء الجداول، نحتاج بيانات اختبارية لنتأكد أن التطبيق يعمل. بدلاً من إدخال البيانات يدوياً واحداً تلو الآخر، نستخدم Seeders.

Seeder هو ملف PHP يضيف بيانات إلى قاعدة البيانات. يشبه "زرع بذور" في الحديقة - تضع البذور (البيانات) في التربة (الجداول) وتنمو (يصبح التطبيق حياً بالبيانات).

إنشاء Seeder:

$ php artisan make:seeder ProjectSeeder

ملاحظة: اسم الـ Seeder يكتب بـ PascalCase (حرف أول كبير) وينتهي بـ "Seeder".

كتابة Seeder:

<?php namespace Database\Seeders; use Illuminate\Database\Seeder; use App\Models\Project; return new class extends Seeder { public function run(): void { Project::create([ // إنشاء مشروع جديد 'user_id' => 1, // المستخدم رقم 1 'name' => 'تطبيق FlowPilot', // اسم المشروع 'description' => 'منصة لإدارة المشاريع', // وصف المشروع 'status' => 'active', // الحالة: نشط 'deadline' => '2025-12-31', // تاريخ التسليم ]); } };

تسجيل Seeder في DatabaseSeeder:

لتشغيل Seeder، يجب تسجيله في الملف الرئيسي DatabaseSeeder.php:

// database/seeders/DatabaseSeeder.php public function run(): void { $this->call([ // استدعاء Seeders UserSeeder::class, // أولاً: المستخدمين ProjectSeeder::class, // ثانياً: المشاريع TaskSeeder::class, // ثالثاً: المهام ]); }

تشغيل Seeders:

$ php artisan db:seed

أو مع تنفيذ Migrations مباشرة: php artisan migrate --seed.

Factories: مصانع البيانات الوهمية

ماذا لو أردنا إضافة 50 مشروعاً بدلاً من مشروع واحد؟ كتابة 50 سطراً في Seeder ستكون متعبة. هنا يأتي دور Factories.

Factory هو "مصنع" ينتج بيانات وهمية (Fake Data) بشكل تلقائي. نستخدم مكتبة Faker (المضمنة في Laravel) لتوليد أسماء، عناوين، تواريخ عشوائية.

إنشاء Factory:

$ php artisan make:factory ProjectFactory

كتابة Factory:

<?php namespace Database\Factories; use App\Models\Project; use Illuminate\Database\Eloquent\Factories\Factory; class ProjectFactory extends Factory { protected $model = Project::class; // ربط المصنع بنموذج Project public function definition(): array // تعريف شكل البيانات { return [ 'name' => fake()->sentence(3), // جملة عشوائية من 3 كلمات كاسم 'description' => fake()->paragraph(2), // فقرتين عشوائيتين كوصف 'status' => fake()->randomElement(['pending', 'active', 'completed']), // عشوائي من القائمة 'deadline' => fake()->dateTimeBetween('now', '+1 year'), // تاريخ عشوائي خلال سنة ]; } }

استخدام Factory في Seeder:

// في ProjectSeeder.php - بدلاً من create واحدة: Project::factory() ->count(20) // أنشئ 20 مشروعاً ->forUser(User::first()) // كلهم للمستخدم الأول ->create(); // احفظهم في قاعدة البيانات

هذا السطر الواحد سينشئ 20 مشروعاً وهمياً بأسماء ووصف وتواريخ عشوائية. السحر!

دوال Faker الشائعة:

الدالة الناتج
fake()->name()اسم عشوائي (مثل: أحمد محمد)
fake()->email()بريد إلكتروني عشوائي
fake()->sentence(3)جملة من 3 كلمات
fake()->paragraph(2)فقرتين من النص
fake()->randomElement(['a','b'])عنصر عشوائي من المصفوفة
fake()->numberBetween(1, 100)رقم عشوائي بين 1 و 100
fake()->date()تاريخ عشوائي

خطوات عملية لبناء قاعدة بيانات FlowPilot

الخطوة 1: إنشاء مشروع Laravel جديد

$ composer create-project laravel/laravel flowpilot
$ cd flowpilot

ملاحظة: إذا كنت تستخدم Laravel مثبتاً مسبقاً، يمكنك تجاهل هذه الخطوة.

الخطوة 2: إعداد قاعدة البيانات

افتح ملف .env في جذر المشروع. ابحث عن هذه الأسطر وغيّرها حسب إعدادات MySQL الخاصة بك:

DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=flowpilot DB_USERNAME=root DB_PASSWORD=

تأكد من إنشاء قاعدة بيانات flowpilot في MySQL (باستخدام phpMyAdmin أو أمر CREATE DATABASE).

الخطوة 3: إنشاء Migrations لجميع الجداول

$ php artisan make:migration create_projects_table
$ php artisan make:migration create_tasks_table
$ php artisan make:migration create_teams_table
$ php artisan make:migration create_project_user_table

الخطوة 4: كتابة كود كل Migration

لكل ملف، اكتب الأعمدة المناسبة كما شرحنا سابقاً. لا تنسَ:

  • id() لكل جدول
  • foreignId(...)->constrained() للمفاتيح الأجنبية
  • timestamps() و softDeletes()
  • nullable() للأعمدة الاختيارية
  • default() للقيم الافتراضية

الخطوة 5: تشغيل الـ Migrations

$ php artisan migrate

تأكد من ظهور جميع الجداول في قاعدة البيانات باستخدام phpMyAdmin أو أي أداة MySQL.

الخطوة 6: إنشاء Seeders و Factories

$ php artisan make:seeder UserSeeder
$ php artisan make:seeder ProjectSeeder
$ php artisan make:seeder TaskSeeder
$ php artisan make:factory ProjectFactory
$ php artisan make:factory TaskFactory

الخطوة 7: زراعة البيانات وتشغيلها

$ php artisan db:seed

أو يمكن استخدام: php artisan migrate:fresh --seed لإعادة إنشاء الجداول مع زراعة البيانات دفعة واحدة.

شرح الكود سطراً بسطر: Migration + Seeder + Factory

1. شرح ملف Migration (create_projects_table.php):

<?php // فتح ملف PHP use Illuminate\Database\Migrations\Migration; // استيراد كلاس Migration الأساسي use Illuminate\Database\Schema\Blueprint; // استيراد كلاس Blueprint (يحدد هيكل الجدول) use Illuminate\Support\Facades\Schema; // استيراد Facade Schema للتعامل مع قاعدة البيانات return new class extends Migration // إنشاء كلاس مجهول (Anonymous Class) يرث من Migration

شرح Schema::create():

Schema::create('projects', function (Blueprint $table) { ... // هنا نحدد أعمدة الجدول });
  • Schema::create: دالة تأخذ اسم الجدول كأول معامل (string)، ووظيفة مجهولة (Closure) كمعامل ثانٍ.
  • Blueprint $table: الكائن الذي يمثل الجدول. نستخدمه لإضافة الأعمدة.

شرح الأعمدة واحداً تلو الآخر:

  • $table->id(): ينشئ عموداً اسمه id من نوع BIGINT UNSIGNED AUTO_INCREMENT. هذا هو المفتاح الأساسي. يرتبط مع PRIMARY KEY في SQL.
  • $table->foreignId('user_id'): ينشئ عموداً اسمه user_id من نوع BIGINT UNSIGNED. هذه هي البداية لإنشاء مفتاح أجنبي.
  • ->constrained(): تخبر Laravel أن هذا العمود هو Foreign Key يشير إلى عمود id في جدول users. Laravel يستنتج اسم الجدول من اسم العمود: user_id → جدول users.
  • ->cascadeOnDelete(): عند حذف مستخدم، تُحذف جميع مشاريعه تلقائياً. هذا يمنع وجود "مشاريع يتيمة" ليس لها مالك.
  • $table->string('name'): ينشئ عموداً من نوع VARCHAR(255). مناسب لأسماء المشاريع.
  • $table->text('description')->nullable(): ينشئ عموداً من نوع TEXT (نص طويل). nullable() يعني أن هذا العمود يمكن أن يكون NULL (فارغاً).
  • $table->string('status')->default('pending'): عمود نصي بقيمة افتراضية 'pending'. عندما يُنشأ مشروع جديد، إن لم نحدد حالة، ستصبح pending تلقائياً.
  • $table->date('deadline')->nullable(): عمود تاريخ (YYYY-MM-DD). اختياري: يمكن أن يكون المشروع بدون تاريخ تسليم.
  • $table->timestamps(): دالة خاصة تنشئ عمودين: created_at و updated_at من نوع TIMESTAMP. Laravel يملؤهما تلقائياً.
  • $table->softDeletes(): ينشئ عموداً اسمه deleted_at من نوع TIMESTAMP. عند "حذف" سجل، نضع التاريخ هنا بدلاً من حذفه فعلياً.

2. شرح Seeder مع Factory:

// في ProjectSeeder.php Project::factory() // نصل إلى Factory المرتبط بنموذج Project ->count(20) // نحدد عدد السجلات: 20 مشروعاً ->create(); // ننفذ عملية الإنشاء والحفظ في قاعدة البيانات

هذا الكود: (1) يأخذ Factory النموذج Project، (2) يحدد أننا نريد 20 نسخة، (3) ينشئها ويحفظها في قاعدة البيانات. كل مشروع سيأخذ قيماً عشوائية من Factory.

3. شرح Factory بالتفصيل:

return [ 'name' => fake()->sentence(3), // fake() يعيد كائن Faker لتوليد بيانات وهمية // sentence(3) يولد جملة من 3 كلمات بالإنكليزية 'description' => fake()->paragraph(2), // paragraph(2) يولد فقرتين 'status' => fake()->randomElement([...]), // يختار قيمة عشوائية من المصفوفة 'deadline' => fake()->dateTimeBetween(...), // تاريخ عشوائي بين تاريخين ];

لاحظ أن Factory لا يحدد user_id. هذا لأننا سنحدده لاحقاً عند استدعاء Factory من Seeder باستخدام forUser().

الأخطاء الشائعة وكيفية حلها

❌ الخطأ 1: "Table already exists" عند تشغيل migrate

السبب: حاولت تشغيل php artisan migrate والجدول موجود بالفعل (ربما من تشغيل سابق).

الحل: استخدم php artisan migrate:fresh لحذف جميع الجداول وإعادة إنشائها. أو php artisan migrate:rollback ثم php artisan migrate.

❌ الخطأ 2: "SQLSTATE[23000]: Integrity constraint violation" (انتهاك المفتاح الأجنبي)

السبب: تحاول إضافة سجل يشير إلى مفتاح أجنبي بقيمة غير موجودة في الجدول الأصلي. مثلاً، تحاول إضافة مشروع user_id = 99 ولا يوجد مستخدم بهذا الرقم.

الحل: تأكد من ترتيب Seeders - أنشئ المستخدمين أولاً، ثم المشاريع. أو استخدم Factory's forUser() الذي يختار مستخدمين موجودين فعلاً.

❌ الخطأ 3: "Class Database\Seeders\XSeeder not found"

السبب: نسيت تسجيل الـ Seeder في DatabaseSeeder.php، أو اسم الـ Seeder خطأ.

الحل: تأكد من إضافة $this->call([XSeeder::class]) في DatabaseSeeder.php، وتأكد من تطابق اسم الملف مع اسم الكلاس.

❌ الخطأ 4: "Unknown column type" أو خطأ في نوع العمود

السبب: استخدام نوع عمود غير موجود في Laravel أو MySQL، أو خطأ إملائي في اسم الدالة (مثلاً stirng بدلاً من string).

الحل: استخدم دالة موجودة. راجع جدول الأنواع في هذا الدرس. أشهر الأخطاء الإملائية: stirng بدلاً من string، foregnId بدلاً من foreignId.

❌ الخطأ 5: فشل الاتصال بقاعدة البيانات "SQLSTATE[HY000] [1045]"

السبب: إعدادات .env غير صحيحة - اسم قاعدة بيانات، اسم مستخدم، أو كلمة مرور خاطئة.

الحل: تأكد من صحة إعدادات DB_DATABASE، DB_USERNAME، DB_PASSWORD في ملف .env. تأكد أن MySQL شغال.

📝 ملخص الدرس

  • 1
    Migrations هي ملفات PHP تصف تغييرات قاعدة البيانات. تسمح بتتبع التغييرات والتراجع عنها.
  • 2
    up() تنفذ التغيير (إنشاء جدول). down() تعكسه (حذف الجدول).
  • 3
    أوامر Artisan: make:migration للإنشاء، migrate للتنفيذ، migrate:rollback للتراجع.
