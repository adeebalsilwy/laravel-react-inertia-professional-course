واجب: إنشاء Migrations و Seeders و Factories لـ FlowPilot

"مخطط البناء (Migration) هو نصف العمل. زراعة البيانات (Seeder) هو النصف الآخر."

نظرة عامة

في هذا الواجب، ستقوم بإنشاء Migrations لجميع جداول FlowPilot، ثم إنشاء Seeders و Factories لزراعة بيانات اختبارية. هذا الواجب يتطلب تنفيذ أوامر Artisan وكتابة كود PHP فعلي.

⏱ الوقت المتوقع: 90-120 دقيقة 💻 يتطلب تشغيل الأوامر في Terminal ⭐ وزن الدرجة: 20% من تقييم الأسبوع

خطوات الواجب التفصيلية

الخطوة 1: إنشاء مشروع Laravel (إن لم يكن موجوداً)

$ composer create-project laravel/laravel flowpilot
$ cd flowpilot

تأكد من إعداد ملف .env للاتصال بقاعدة البيانات flowpilot.

الخطوة 2: إنشاء 4 ملفات Migration

$ php artisan make:migration create_projects_table
$ php artisan make:migration create_tasks_table
$ php artisan make:migration create_teams_table
$ php artisan make:migration create_project_user_table

تأكد من ظهور 4 ملفات جديدة في database/migrations/.

الخطوة 3: كتابة كود Migration لجدول projects

افتح ملف create_projects_table.php واكتب الكود التالي داخل up():

Schema::create('projects', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->string('name');
    $table->text('description')->nullable();
    $table->string('status')->default('pending');
    $table->date('deadline')->nullable();
    $table->timestamps();
    $table->softDeletes();
});
// في down(): Schema::dropIfExists('projects');

الخطوة 4: كتابة كود Migration لجدول tasks

في create_tasks_table.php، أضف الأعمدة: id, project_id (FK), assigned_to (FK, nullable), title, description (nullable), status (default 'pending'), priority (enum: low, medium, high, critical, default 'medium'), due_date (nullable), timestamps, softDeletes.

الخطوة 5: كتابة Migration لجدول teams و project_user

جدول teams: id, owner_id (FK → users), name, description (nullable), timestamps.

جدول project_user (Pivot): id, project_id (FK), user_id (FK), role (nullable), joined_at (nullable), timestamps.

الخطوة 6: إنشاء Seeders

$ php artisan make:seeder UserSeeder
$ php artisan make:seeder ProjectSeeder
$ php artisan make:seeder TaskSeeder

في UserSeeder: استخدم User::create() لإنشاء 3 مستخدمين (أسماء عربية: أحمد، سارة، محمد).

الخطوة 7: إنشاء Factories

$ php artisan make:factory ProjectFactory
$ php artisan make:factory TaskFactory

في ProjectFactory: استخدم fake()->sentence(3) للاسم، fake()->paragraph(2) للوصف، fake()->randomElement() للحالة، fake()->dateTimeBetween() للتاريخ.

الخطوة 8: ربط كل شيء في DatabaseSeeder

في DatabaseSeeder.php:

$this->call([
    UserSeeder::class,
    ProjectSeeder::class,
    TaskSeeder::class,
]);

في ProjectSeeder، استخدم الـ Factory لإنشاء 10 مشاريع للمستخدم الأول.

الخطوة 9: تنفيذ كل شيء

$ php artisan migrate:fresh --seed

يجب أن تظهر رسالة نجاح. تحقق من قاعدة البيانات أن الجداول والبيانات موجودة.

الخطوة 10: التحقق والتصدير

استخدم php artisan migrate:status للتأكد من تنفيذ جميع Migrations. قم بعمل تصدير (Screenshot) لـ phpMyAdmin يظهر الجداول مع البيانات.

⭐ التحدي الإضافي

  1. أضف 3 حقول إضافية في جدول project_user: hourly_rate (decimal), is_manager (boolean), notes (text).
  2. أنشئ Seeder يضيف 5 أعضاء لكل مشروع بشكل عشوائي.
  3. استخدم fake()->imageUrl() في ProjectFactory لإضافة رابط صورة وهمي.
  4. أنشئ Migration رابع لإضافة عمود color (string) إلى جدول projects بعد إنشائه. استخدم php artisan make:migration add_color_to_projects_table.

📤 دليل التسليم

  1. قم بتصدير ملف SQL لقاعدة البيانات (من phpMyAdmin أو mysqldump)
  2. خذ لقطة شاشة تظهر جداول قاعدة البيانات في phpMyAdmin
  3. خذ لقطة شاشة لأمر php artisan migrate:status
  4. ضع جميع الملفات (Migrations, Seeders, Factories) و الصور في مجلد مضغوط باسم week-03-lesson-02-yourname.zip
  5. ارفع على Google Drive وأرسل الرابط
