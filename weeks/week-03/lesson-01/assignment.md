# واجب الدرس 9: تصميم قاعدة بيانات FlowPilot قبل كتابة الجداول

## الهدف من الواجب
تطبيق مفاهيم تصميم قواعد البيانات (الجداول، الأعمدة، العلاقات، المفاتيح) على مشروع FlowPilot بشكل عملي. هذا الواجب لا يتطلب كتابة أي كود Laravel - فقط تخطيط وتصميم على الورق أو في مستند.

## المهارات المستهدفة
- تحديد الكيانات (Entities) لتطبيق حقيقي
- تصميم الجداول مع اختيار أنواع البيانات الصحيحة
- تحديد العلاقات بين الجداول (One-to-Many, Many-to-Many)
- رسم مخطط ERD
- فهم متى نستخدم جدول Pivot

## المطلوب بالتفصيل

### 1. قائمة الكيانات
اكتب 5-7 كيانات يحتاجها تطبيق FlowPilot. مثال:
- User (المستخدم)
- Project (المشروع)
- Task (المهمة)
- Team (الفريق)
- Comment (التعليق)
- Attachment (الملف المرفق)
- Notification (الإشعار)

لكل كيان، اكتب جملة تصف دوره.

### 2. تصميم جدول Projects
ارسم جدولاً بـ 8 أعمدة على الأقل مع:
- اسم العمود بالإنجليزية
- نوع البيانات (string, text, foreignId, enum, date, boolean, timestamps)
- هل هو Required أم Nullable؟
- القيمة الافتراضية (Default) إن وجدت
- شرح مبسط للعمود

### 3. تصميم جدول Tasks
ارسم جدول Tasks مع المفاتيح الأجنبية:
- project_id (FK → projects)
- assigned_to (FK → users)
- title, description, status, priority, due_date
- timestamps(), softDeletes()

### 4. العلاقات
حدد نوع العلاقة لكل زوج:
- User ↔ Projects
- Projects ↔ Tasks
- Projects ↔ Users (members)
- Tasks ↔ Users (assignee)
- Teams ↔ Users (owner)

### 5. مخطط ERD
ارسم مخططاً كاملاً يحتوي على 5+ جداول مع:
- مستطيل لكل جدول
- أعمدة رئيسية (PK, FK, name, status, timestamps)
- خطوط العلاقات مع تحديد النوع
- وسيلة إيضاح (Legend)

### 6. شرح كتابي (اختياري للمتفوقين)
في ملف explanation.md، اجب على:
- لماذا اخترت هذا التصميم؟
- ما التحديات التي واجهتها؟
- كيف يمكن تحسين التصميم مستقبلاً؟

## معايير القبول (20 درجة)
- اكتمال الكيانات: 3 درجات
- صحة الأعمدة وأنواع البيانات: 4 درجات
- صحة العلاقات: 5 درجات
- مخطط ERD كامل وواضح: 4 درجات
- الشرح والتوثيق: 2 درجات
- التحدي الإضافي (Pivot Table): 2 درجات إضافية

## التحدي الإضافي
- صمم جدول Pivot project_user مع أعمدة إضافية (role, joined_at)
- أضف جدول Comments مع علاقاته
- أضف جدول Attachments

## ملاحظات
- هذا واجب تخطيطي فقط. لا تكتب أي كود Laravel أو SQL
- يمكنك استخدام ورقة وقلم أو برامج رسم مثل draw.io
- المخطط يجب أن يكون مفهوماً لشخص آخر
