Eloquent Models: التحدث مع قاعدة البيانات بلغة PHP

"بدلاً من كتابة استعلامات SQL المعقدة، لنتحدث مع جداولنا كأنها كائنات برمجية بسيطة."

ما هو الـ Model؟

الـ Model هو ملف PHP يمثل جدولاً معيناً في قاعدة البيانات. إذا كان لدينا جدول اسمه projects، سيكون لدينا Model اسمه Project.

لماذا نستخدمه؟ لأن Laravel يوفر أداة عبقرية تسمى Eloquent. بفضلها، لا تحتاج لكتابة SQL؛ بل تكتب Project::all() وسيفهم Laravel تلقائياً أنك تريد "كل السجلات من جدول المشاريع".

العلاقات (Relationships)

في FlowPilot، البيانات مرتبطة ببعضها. هناك أنواع عديدة من العلاقات، أهمها:

  • One to Many (واحد لمتعدد): المشروع الواحد يحتوي على "مهام كثيرة".
  • Many to Many (متعدد لمتعدد): المهمة الواحدة يمكن أن يعمل عليها "أعضاء كثيرون"، والعضو الواحد يمكن أن يعمل على "مهام كثيرة".

مثال لتقريب الفكرة (العائلة)

تخيل الأب هو "المشروع" و الأبناء هم "المهام".

- الأب الواحد يمكن أن يكون له أبناء كثيرون (One to Many).
- كل ابن ينتمي لأب واحد محدد (Belongs To).

في الكود، عندما تطلب $project->tasks، أنت تطلب من Laravel: "أحضر لي كل أبناء هذا الأب".

التطبيق العملي

الخطوة 1: إنشاء Model للمشروع

php artisan make:model Project

الخطوة 2: تعريف العلاقة مع المهام

داخل ملف app/Models/Project.php:

public function tasks() {

  return $this->hasMany(Task::class);

}
