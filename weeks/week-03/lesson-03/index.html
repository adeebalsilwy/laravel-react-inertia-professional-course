فهم Eloquent Models والعلاقات بين البيانات

"Eloquent هو المترجم الذي يسمح لك بالتحدث مع قاعدة البيانات بلغة PHP سهلة ومفهومة."

ما هو Eloquent ORM؟

Eloquent هو ORM (Object-Relational Mapping) - تقنية تترجم البيانات بين قاعدة البيانات (SQL) ولغة PHP (OOP). ببساطة: هو مترجم بشري يتحدث مع MySQL نيابة عنك.

بدون Eloquent، كنا سنكتب استعلامات SQL مثل: SELECT * FROM projects WHERE user_id = 1. مع Eloquent، نكتب: User::find(1)->projects. الفرق واضح!

كل Model في Laravel يمثل جدولاً في قاعدة البيانات. كل instance (كائن) من الـ Model يمثل سجلاً (Row) واحداً. كل خاصية (Property) في الـ Model تمثل عموداً (Column).

🎯 قاعدة ذهبية:

Model = Table | Object = Row | Property = Column

مثال: Project Model يمثل جدول projects. $project->name يقرأ قيمة عمود name من ذلك السجل.

إنشاء Models باستخدام Artisan

# إنشاء Model مع الـ Migration تلقائياً
$ php artisan make:model Project -m

الـ -m تخبر Artisan بإنشاء ملف Migration مع الـ Model. يمكنك أيضاً إنشاء Model + Migration + Controller + Seeder معاً بـ -mcs.

مثال لملف Model (Project.php):

<?php namespace App\Models; use Illuminate\Database\Eloquent\Model; use Illuminate\Database\Eloquent\SoftDeletes; class Project extends Model { use SoftDeletes; // تفعيل الحذف الناعم protected $fillable = [ // الحقول المسموح بملئها (للحماية) 'user_id', 'name', 'description', 'status', 'deadline', ]; }

شرح الكود:

  • namespace App\Models: مجلد الـ Models (Laravel 12 ينشئها هنا بدلاً من App).
  • extends Model: يرث من كلاس Eloquent Model الأساسي، مما يمنحه كل القدرات.
  • use SoftDeletes: Trait يضيف دوال الحذف الناعم (مثل trash(), restore()).
  • $fillable: يسمح فقط للأعمدة المذكورة أن تُملأ عبر create() أو update().

$fillable و $guarded: حماية البيانات (Mass Assignment)

تخيل أن لديك Model المستخدم، وهناك مهاجم يرسل طلباً HTTP يحتوي على is_admin = true. بدون حماية، سيتحول المستخدم إلى مشرف! لمنع هذا، نستخدم Mass Assignment Protection.

$fillable = القائمة البيضاء: الحقول المسموح فقط بملئها.

$guarded = القائمة السوداء: الحقول الممنوعة من الملء.

// السماح فقط بهذه الحقول: protected $fillable = ['name', 'email']; // أو منع هذه الحقول فقط (والباقي مسموح): protected $guarded = ['is_admin'];

💡 أفضل ممارسة:

استخدم $fillable وحدد الأعمدة المسموحة. هذا أكثر أماناً من $guarded.

تعريف العلاقات (Relationships) في Models

هذا هو أهم جزء في هذا الدرس. العلاقات تجعل Eloquent قوياً جداً. سنتعلم 3 أنواع أساسية:

belongsTo

ينتمي إلى (المفتاح الأجنبي هنا)

hasMany

يملك عدة (المفتاح الأجنبي في الجدول الآخر)

belongsToMany

علاقة متعددة لمتعددة (Pivot)

1. belongsTo (ينتمي إلى)

تستخدم عندما يكون عندك المفتاح الأجنبي وتريد الوصول للسجل الأصلي.

// في Project Model - المشروع ينتمي لمستخدم واحد public function user() { return $this->belongsTo(User::class); }

الآن: $project->user يعيد كائن User (صاحب المشروع).

2. hasMany (يملك عدة)

تستخدم عندما لا يوجد عندك المفتاح الأجنبي، بل هو في الجدول الآخر.

// في User Model - المستخدم يملك عدة مشاريع public function projects() { return $this->hasMany(Project::class); }

الآن: $user->projects يعيد مجموعة من Projects (جميع مشاريع المستخدم).

// في Project Model - المشروع يحتوي على عدة مهام public function tasks() { return $this->hasMany(Task::class); }

3. belongsToMany (متعدد لمتعدد)

تستخدم للعلاقات Many-to-Many مع جدول Pivot.

// في Project Model - المشروع يضم عدة أعضاء (Users) public function members() { return $this->belongsToMany(User::class) ->withPivot('role', 'joined_at') // أعمدة إضافية من جدول Pivot ->withTimestamps(); }

الآن: $project->members يعيد مجموعة Users (أعضاء المشروع). $project->members[0]->pivot->role يقرأ الدور من جدول Pivot.

// وفي User Model - المستخدم عضو في عدة مشاريع public function projects() // لاحظ: هذا مختلف عن hasMany projects { return $this->belongsToMany(Project::class); }

⚠️ في User، سنحتاج تسمية مختلفة للعلاقة Many-to-Many (مثلاً memberOfProjects) للتمييز عن hasMany projects.

الاستعلام بـ Eloquent: ::all(), ::find(), ::where(), ::with()

Eloquent يوفر دوالاً سهلة للاستعلام عن قاعدة البيانات. إليك أهمها:

الدالة الاستخدام مثال
::all()جلب جميع السجلات$projects = Project::all();
::find($id)جلب سجل واحد بالـ ID$project = Project::find(1);
::findOrFail($id)جلب أو خطأ 404$project = Project::findOrFail(1);
::where()استعلام بشرطProject::where('status', 'active')->get();
::with()Eager Loading (جلب العلاقات)Project::with('tasks')->get();
->get()تنفيذ الاستعلام وجلب النتائج...->where('status','active')->get();
->first()جلب أول سجل فقطProject::where('status','active')->first();
->count()عدد السجلاتProject::where('status','active')->count();
->pluck()جلب عمود واحد كمصفوفةProject::pluck('name'); // ['مشروع 1', 'مشروع 2']

أمثلة عملية من FlowPilot:

// جلب جميع المشاريع النشطة $activeProjects = Project::where('status', 'active')->get(); // جلب أول 10 مشاريع مرتبة بالأحدث $recentProjects = Project::latest()->take(10)->get(); // جلب مشروع معين مع جميع مهامه ومستخدمه $project = Project::with(['user', 'tasks'])->findOrFail($id); // جلب عدد المهام لكل مشروع $projectCount = Project::withCount('tasks')->get(); // الآن $project->tasks_count يعطي العدد // جلب المشاريع التي ليس لها مهام $emptyProjects = Project::doesntHave('tasks')->get();

Accessors و Mutators (تعديل البيانات عند القراءة/الكتابة)

Accessor = دالة تُعدّل قيمة العمود عند القراءة. مثلاً: تحويل تاريخ الميلاد إلى صيغة معينة.

Mutator = دالة تُعدّل قيمة العمود عند الكتابة. مثلاً: تحويل كلمة المرور إلى Hash قبل الحفظ.

مثال Accessor في Project Model:

// عند قراءة status، نترجمها إلى العربية public function getStatusAttribute($value) { return [ 'pending' => 'قيد الانتظار', 'active' => 'نشط', 'completed' => 'مكتمل', ][$value] ?? $value; }

الآن: $project->status سيعيد "نشط" بدلاً من "active". ملاحظة: التسمية getXAttribute حيث X هو اسم العمود بـ PascalCase.

مثال Mutator في User Model:

// عند حفظ كلمة المرور، نستخدم Hash تلقائياً public function setPasswordAttribute($value) { $this->attributes['password'] = bcrypt($value); }

الآن: User::create(['password' => 'secret']) سيحفظ $2y$10... بدلاً من 'secret'.

مشكلة N+1 (الفخ الخفي للمبتدئين)

مشكلة N+1 هي واحدة من أشهر مشاكل الأداء في Laravel. تخيل هذا السيناريو:

// الكود التالي يبدو بريئاً... $projects = Project::all(); // استعلام واحد: SELECT * FROM projects foreach ($projects as $project) { echo $project->user->name; // استعلام لكل مشروع: SELECT * FROM users WHERE id = ? }

إذا كان لديك 100 مشروع، سيتم تنفيذ 101 استعلام! واحد لجلب المشاريع، و100 لجلب المستخدم (لكل مشروع على حدة). هذا هو N+1.

الحل: Eager Loading باستخدام with()

$projects = Project::with('user')->get(); // استعلامان فقط! // 1. SELECT * FROM projects // 2. SELECT * FROM users WHERE id IN (1, 2, 3, ...)

بدلاً من 101 استعلام، أصبح لدينا 2 فقط. هذا هو الفرق بين تطبيق بطيء وتطبيق سريع.

نماذج FlowPilot الكاملة مع العلاقات

1. User Model

<?php namespace App\Models; use Illuminate\Database\Eloquent\Model; class User extends Model { // المشاريع التي يملكها المستخدم (hasMany = هو مالك) public function ownedProjects() { return $this->hasMany(Project::class, 'user_id'); } // المشاريع التي هو عضو فيها (belongsToMany = عضو فريق) public function memberOfProjects() { return $this->belongsToMany(Project::class) ->withPivot('role', 'joined_at') ->withTimestamps(); } // المهام المسندة لهذا المستخدم public function tasks() { return $this->hasMany(Task::class, 'assigned_to'); } }

3. Task Model

class Task extends Model { use SoftDeletes; protected $fillable = ['project_id', 'assigned_to', 'title', 'description', 'status', 'priority', 'due_date']; // المهمة تنتمي لمشروع public function project() { return $this->belongsTo(Project::class); } // المهمة مسندة لمستخدم (اختياري) public function assignee() { return $this->belongsTo(User::class, 'assigned_to'); } }

الأخطاء الشائعة في Eloquent Models

❌ الخطأ 1: "Column not found: 1054 Unknown column 'user_id'"

السبب: علاقة belongsTo تتوقع وجود عمود user_id ولكن لا يوجد في جدول المشاريع.

الحل: تأكد من أن الـ Migration يحتوي على $table->foreignId('user_id')->constrained(). أعد تشغيل الـ Migrations.

❌ الخطأ 2: "Call to undefined method ..." عند استخدام العلاقات

السبب: نسيت تعريف الدالة في الـ Model. مثلاً، استخدمت $project->tasks ولكن لا توجد دالة tasks() في Project Model.

الحل: أضف العلاقة المفقودة إلى الـ Model: public function tasks() { return $this->hasMany(Task::class); }.

❌ الخطأ 3: N+1 Query Problem (تطبيق بطيء)

السبب: جلب البيانات دون with()، مما يسبب استعلامات متعددة في حلقة foreach.

الحل: استخدم ::with('user', 'tasks')->get() لجلب العلاقات مقدماً.

❌ الخطأ 4: "MassAssignmentException" (لا يمكن حفظ البيانات)

السبب: لم تحدد $fillable في الـ Model. Eloquent يمنع ملء جميع الحقول للحماية.

الحل: أضف protected $fillable = ['name', 'description', ...]; في الـ Model.

❌ الخطأ 5: علاقة عكسية خاطئة (hasMany مكان belongsTo)

السبب: استخدام hasMany في Task (بينما Task يحتوي على FK project_id). الصحيح هو belongsTo.

الحل: من يملك FK (المفتاح الأجنبي) يستخدم belongsTo. الطرف الآخر يستخدم hasMany.
