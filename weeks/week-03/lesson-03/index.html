الـ Models: كيف نجعل الجداول "تتحدث" لغة البرمجة؟

"الـ Model هو السفير الذي يربط بين كود PHP وبين صفوف قاعدة البيانات."

ما هو الـ Model؟

الجدول في قاعدة البيانات هو مجرد بيانات جافة. لكي نستطيع استخدامه في الكود (مثلاً: حذف مشروع أو تعديل اسم مستخدم)، نحتاج لملف PHP يمثل هذا الجدول. هذا الملف نسميه Model.

في Laravel، نستخدم تقنية مذهلة تسمى Eloquent. بفضلها، لن تحتاج لكتابة أوامر SQL معقدة؛ بل ستكتب كوداً بسيطاً مثل Project::all() لجلب كل المشاريع!

العلاقات (Relationships)

في FlowPilot، البيانات مترابطة.
- المشروع الواحد له "مهام كثيرة" (HasMany).
- المهمة الواحدة "تنتمي لمشروع واحد" (BelongsTo).
تعلم هذه العلاقات سيجعل تطبيقك ذكياً جداً في جلب البيانات المرتبطة ببعضها.

التطبيق العملي

1

إنشاء موديل المشروع

php artisan make:model Project

افتح ملف app/Models/Project.php وأضف علاقة المهام:

public function tasks() {

  return $this->hasMany(Task::class);

}
