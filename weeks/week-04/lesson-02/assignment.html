واجب الدرس 14: تطبيق Policies في FlowPilot

"حماية بيانات المستخدمين هي مسؤوليتنا كمطورين. اليوم نتعلم كيف نجعل كل مستخدم يتحكم فقط في مشاريعه."

01 الهدف من الواجب

إنشاء ProjectPolicy تمنع المستخدمين من تعديل أو حذف مشاريع بعضهم البعض، مع السماح لأعضاء الفريق برؤية المشروع. سنطبق هذا باستخدام php artisan make:policy و authorize() ونتأكد من أن التحقق يتم في الخلفية والأمامية.

02 خطوات التنفيذ بالتفصيل

1. إنشاء Policy لموديل Project

تأكد أنك داخل مجلد مشروع FlowPilot، ثم نفذ:

php artisan make:policy ProjectPolicy --model=Project

📌 الملف الناتج: app/Policies/ProjectPolicy.php

2. تسجيل Policy في AuthServiceProvider

افتح app/Providers/AuthServiceProvider.php وأضف:

use App\Models\Project;
use App\Policies\ProjectPolicy;

protected $policies = [
Project::class => ProjectPolicy::class,
];

3. كتابة دوال Policy كاملة

افتح app/Policies/ProjectPolicy.php واكتب الدوال التالية:

public function view(User $user, Project $project): bool
{
// المالك أو عضو الفريق يمكنه رؤية المشروع
if ($user->id === $project->user_id) return true;
return $project->members()->where('user_id', $user->id)->exists();
}

public function create(User $user): bool
{
return true; // أي مستخدم مسجل يمكنه إنشاء مشروع
}

public function update(User $user, Project $project): bool
{
return $user->id === $project->user_id;
}

public function delete(User $user, Project $project): bool
{
return $user->id === $project->user_id;
}

4. استخدام authorize() في ProjectController

أضف $this->authorize() قبل كل عملية في app/Http/Controllers/ProjectController.php:

public function show(Project $project)
{
$this->authorize('view', $project);
return Inertia::render('Projects/Show', ['project' => $project]);
}

public function update(Request $request, Project $project)
{
$this->authorize('update', $project);
// ... validation and update
}

public function destroy(Project $project)
{
$this->authorize('delete', $project);
$project->delete();
return redirect()->route('projects.index');
}

5. اختبار صلاحيات المشاهدة

أنشئ مستخدمين اثنين (مثلاً: ahmed@test.com و sara@test.com). سجل بأحمد وأنشئ مشروعاً. ثم سجل الخروج وسجل بسارة وحاول فتح مشروع أحمد. يجب أن تظهر رسالة 403.

6. إخفاء الأزرار في واجهة React

في resources/js/Pages/Projects/Show.jsx أضف شرطاً لإخفاء أزرار التعديل والحذف عن المستخدمين غير المالكين:

const { props } = usePage();
const user = props.auth.user;
const isOwner = user?.id === project.user_id;

{isOwner && (
<Link href={`/projects/${project.id}/edit`}>تعديل</Link>
)}

📋 معايير التقييم

  • ✅ إنشاء ProjectPolicy وتسجيله في AuthServiceProvider.
  • ✅ كتابة دوال view, create, update, delete بشكل صحيح.
  • ✅ استخدام $this->authorize() في جميع دوال Controller.
  • ✅ المستخدم لا يمكنه تعديل أو حذف مشروع ليس مالكه.
  • ✅ أزرار التعديل والحذف تختفي للمستخدمين غير المالكين في React.
  • ✅ ظهور رسالة 403 عند محاولة الوصول غير المصرح به.

🚀 تحدي إضافي: إضافة صلاحية المشرف

عدّل delete() في ProjectPolicy ليسمح لمستخدمين يحملون خاصية is_admin = true بحذف أي مشروع. أضف عمود is_admin إلى جدول users عبر Migration جديد.
