📝 كويز الدرس 14: Policies والصلاحيات

"اختبر فهمك لكيفية حماية مشاريع المستخدمين من الوصول غير المصرح به."

عدد الأسئلة: 9 (5 MCQ + 2 T/F + 1 نظري + 1 عملي)

1. ما الفرق بين Authentication و Authorization؟

✅ الشرح: Authentication = من أنت؟ (البريد + كلمة السر). Authorization = ماذا يسمح لك أن تفعل؟ (صلاحيات التعديل/الحذف).

2. أي أمر ينشئ Policy مرتبطة بموديل Project؟

✅ الشرح: php artisan make:policy ProjectPolicy --model=Project ينشئ Policy مع دوال افتراضية مرتبطة بموديل Project.

3. أين يتم تسجيل Policy لربطها بالموديل؟

✅ الشرح: Policy تسجل في AuthServiceProvider.php عبر مصفوفة $policies التي تربط كل Model بـ Policy.

4. ماذا تفعل $this->authorize('update', $project) في Controller؟

✅ الشرح: authorize() تستدعي الدالة المناسبة من Policy. إذا رجعت false، ترسل خطأ 403 تلقائياً.

5. لماذا نتحقق من الصلاحيات في React أيضاً وليس فقط في Laravel؟

✅ الشرح: التحقق في Laravel هو الأساس والأهم. التحقق في React يحسن تجربة المستخدم بإخفاء الأزرار غير المسموح بها.

6. (صح/خطأ) الدالة create() في Policy لا تحتاج لكائن (Object) من الموديل.

✅ الشرح: صح. لأن create يتعلق بإنشاء كائن جديد (لم يتم إنشاؤه بعد)، لذا تأخذ فقط User كمعامل.

7. (صح/خطأ) التحقق من الصلاحيات في الواجهة الأمامية (React) يغني عن التحقق في الخلفية (Laravel).

✅ الشرح: خطأ. التحقق في الخلفية (Laravel) هو خط الدفاع الأول والأهم. الواجهة الأمامية يمكن اختراقها بسهولة.

8. (نظري) اشرح بمثال: لماذا نحتاج Policy منفصلة لكل موديل (Project, Task, Comment) بدلاً من Policy واحدة لكل شيء؟

💡 الإجابة النموذجية: كل موديل له قواعد صلاحيات مختلفة. مثلاً، صلاحية تعديل مشروع تختلف عن صلاحية تعديل تعليق. المشروع: يعدله المالك فقط. التعليق: يعدله كاتبه أو مشرف المشروع. Policy منفصلة لكل موديل تنظم الكود وتجعل كل Policy مسؤولة عن صلاحيات موديل واحد فقط (Single Responsibility Principle).

9. (عملي) أكتب دالة Policy (update) تسمح للمالك فقط بتعديل المشروع، وللمشرف (admin) أيضاً.

💡 الإجابة النموذجية:
public function update(User $user, Project $project): bool
{
// المالك يسمح له بالتعديل
if ($user->id === $project->user_id) {
return true;
}
// المشرف (admin) أيضاً يسمح له
if ($user->is_admin) {
return true;
}
return false;
}
