# ملخص الدرس: Authorization Policies

## الفكرة الأساسية
Authorization يجيب عن سؤال: هل المستخدم المعرّف مسموح له بتنفيذ هذا الإجراء؟ قد يكون المستخدم مسجلاً دخوله، لكنه لا يملك حق تعديل مشروع شخص آخر.

## لماذا هذا مهم؟
المصادقة وحدها لا تكفي. نحتاج صلاحيات تمنع الوصول غير المصرح للبيانات والعمليات الحساسة.

## داخل FlowPilot
في FlowPilot يستطيع صاحب المشروع التعديل، وقد يستطيع عضو الفريق التعليق، لكن لا يستطيع مستخدم غريب حذف المشروع.

## المثال
public function update(User $user, Project $project): bool
{
    return $project->user_id === $user->id;
}

## نتيجة الدرس
سياسات صلاحيات تحمي المشاريع والمهام.
