واجب الدرس 13: تفعيل نظام المصادقة في FlowPilot

"اليوم ستضع أول قفل ذكي على مشروعك. هذا الواجب سيعلمك كيف تجعل التطبيق يفرق بين الزوار وأعضاء الفريق."

01 الهدف من الواجب

تمكين المستخدم من إنشاء حساب جديد (Register) وتسجيل الدخول (Login) إلى لوحة تحكم FlowPilot باستخدام Laravel Breeze مع React و Inertia.js. في نهاية هذا الواجب، سيكون لديك نظام مصادقة كامل يعمل بكفاءة.

02 خطوات التنفيذ بالتفصيل

1. تثبيت حزمة Breeze

افتح Terminal (PowerShell أو CMD) داخل مجلد مشروع FlowPilot ونفذ الأمر:

composer require laravel/breeze --dev

📌 الملفات المتأثرة: سيتم إضافة حزمة Breeze إلى composer.json وتحميلها إلى مجلد vendor/.

2. تثبيت Breeze مع React + Inertia

php artisan breeze:install react

📌 الملفات المتأثرة:

  • app/Http/Controllers/Auth/ - جميع Controllers المصادقة (Login, Register, Password Reset)
  • resources/js/Pages/Auth/ - صفحات React (Login.jsx, Register.jsx, ForgotPassword.jsx, ResetPassword.jsx)
  • routes/auth.php - مسارات المصادقة
  • resources/js/Components/ - مكونات React المشتركة (Input, Button, ValidationErrors)

3. تثبيت مكتبات JavaScript (NPM)

npm install

📌 الملفات المتأثرة: node_modules/ (مكتبات: React, React DOM, Inertia.js, Tailwind CSS, Axios).

4. إعداد ملف البيئة (.env)

افتح ملف .env في جذر المشروع وتأكد من صحة إعدادات قاعدة البيانات:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=flowpilot
DB_USERNAME=root
DB_PASSWORD=

📌 إذا كنت تستخدم XAMPP، اترك كلمة المرور فارغة. إذا كنت تستخدم MAMP أو Laragon، عدّل حسب إعداداتك.

5. تشغيل التهجير (Migration)

php artisan migrate

📌 هذا الأمر ينشئ الجداول في قاعدة البيانات: users، password_reset_tokens، personal_access_tokens، sessions.

6. تشغيل خادم التطوير

افتح نافذتي Terminal:

# النافذة 1: خادم Laravel
php artisan serve

# النافذة 2: خادم Vite (JavaScript)
npm run dev

7. إنشاء حساب جديد (اختبار)

افتح المتصفح واذهب إلى http://127.0.0.1:8000/register.

املأ البيانات التالية:

  • Name: أحمد السيد
  • Email: ahmed@flowpilot.com
  • Password: password123 (8 أحرف على الأقل)
  • Confirm Password: password123

8. التحقق من النتيجة

بعد الضغط على "Register"، تأكد من:

  • ✅ تم توجيهك إلى صفحة /dashboard.
  • ✅ يظهر اسم المستخدم "أحمد السيد" في أعلى الصفحة.
  • ✅ تظهر رسالة ترحيبية (إن وجدت).

9. اختبار تسجيل الخروج والدخول مرة أخرى

اضغط على زر "Logout" (تسجيل الخروج)، ثم حاول فتح /dashboard مباشرة. يجب أن يتم تحويلك إلى صفحة تسجيل الدخول. سجل الدخول مرة أخرى بالبيانات نفسها وتأكد من نجاح الدخول.

10. [تحدي إضافي] تخصيص واجهة FlowPilot

افتح ملف resources/js/Pages/Auth/Login.jsx وقم بالتالي:

  • غيّر عنوان الصفحة من "Log in" إلى "تسجيل الدخول إلى FlowPilot".
  • غيّر لون الزر الرئيسي من bg-gray-900 إلى bg-indigo-600.
  • أضف شعار FlowPilot (نص "FlowPilot" بخط كبير) أعلى النموذج.

📂 الملفات التي يجب مراجعتها

routes/auth.php

مسارات المصادقة (Login, Register, Logout, Password Reset)

app/Models/User.php

نموذج المستخدم مع دوال المصادقة

resources/js/Pages/Auth/Login.jsx

صفحة تسجيل الدخول (React)

resources/js/Pages/Auth/Register.jsx

صفحة إنشاء حساب جديد (React)

app/Http/Middleware/HandleInertiaRequests.php

مشاركة بيانات المستخدم مع React

routes/web.php

مسارات التطبيق الرئيسية (مع or بدون middleware)

📋 معايير التقييم

  • ✅ نجاح عملية التسجيل (Register) بدون أخطاء.
  • ✅ نجاح عملية تسجيل الدخول (Login) بالبيانات المسجلة.
  • ✅ ظهور اسم المستخدم في شريط التنقل العلوي بعد الدخول.
  • ✅ تحويل الزوار غير المسجلين إلى صفحة Login عند محاولة فتح /dashboard.
  • ✅ القدرة على تسجيل الخروج والعودة بنجاح.
  • ✅ [تحدي إضافي] تخصيص ألوان ونصوص واجهة الدخول لتناسب هوية FlowPilot.

🚀 تحدي إضافي: قم ببناء صفحه "مرحباً بك"

بعد نجاح المصادقة، قم بإنشاء صفحة Dashboard بسيطة في React تعرض:
1. اسم المستخدم المسجل.
2. بريده الإلكتروني.
3. تاريخ انضمامه (created_at).
استخدم usePage().props.auth.user لجلب هذه البيانات.
