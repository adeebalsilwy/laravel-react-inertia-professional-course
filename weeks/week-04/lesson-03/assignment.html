واجب الدرس 15: تطبيق Middleware في FlowPilot

"اليوم نضيف طبقة حماية إضافية. كل طلب سيمر عبر نقطة تفتيش تتأكد من هوية المستخدم وصلاحياته."

01 الهدف من الواجب

إنشاء Middleware مخصص اسمه CheckProjectAccess يمنع المستخدمين من الوصول إلى مشاريع لا يملكونها. سنطبقه على مسارات المشاريع في FlowPilot بالإضافة إلى auth Middleware.

02 خطوات التنفيذ بالتفصيل

1. إنشاء Middleware مخصص

php artisan make:middleware CheckProjectAccess

📌 الملف الناتج: app/Http/Middleware/CheckProjectAccess.php

2. كتابة منطق الفحص في Middleware

افتح الملف واكتب الكود التالي داخل الدالة handle():

$user = $request->user();
$projectId = $request->route('project');
$project = Project::find($projectId);

if (!$project) {
abort(404, 'المشروع غير موجود');
}

if ($user->id !== $project->user_id &&
!$project->members()->where('user_id', $user->id)->exists()) {
abort(403, 'ليس لديك صلاحية الوصول لهذا المشروع');
}

return $next($request);

📌 تذكر: أضف use App\Models\Project; في أعلى الملف.

3. تسجيل Middleware في Kernel.php

افتح app/Http/Kernel.php وأضف السطر التالي في مصفوفة $middlewareAliases:

'check.project' => \App\Http\Middleware\CheckProjectAccess::class,

4. تطبيق Middleware على مسارات المشاريع

في routes/web.php:

Route::middleware(['auth'])->group(function () {
Route::resource('projects', ProjectController::class)
->middleware(['check.project']);
});

5. اختبار Middleware

أنشئ مستخدمين اثنين (أحمد وسارة). سجل بأحمد وأنشئ مشروعاً. سجل الخروج وسجل بسارة. حاول فتح /projects/1 (مشروع أحمد). يجب أن تظهر رسالة 403 Forbidden.

6. (تحدي) Middleware مع Parameter للتحقق من الدور

أنشئ Middleware باسم CheckRole يمكنه استقبال أدوار مسموح بها:

// إنشاء Middleware
php artisan make:middleware CheckRole

// في CheckRole.php:
public function handle(Request $request, Closure $next, ...$roles)
{
if (!in_array($request->user()->role, $roles)) {
abort(403);
}
return $next($request);
}

// التسجيل في Kernel:
'role' => \App\Http\Middleware\CheckRole::class,

// الاستخدام:
Route::get('/admin', [AdminController::class, 'index'])
->middleware('role:admin,manager');

app/Http/Kernel.php

تطبيق middleware على المسارات

📋 معايير التقييم

  • ✅ إنشاء CheckProjectAccess Middleware.
  • ✅ تسجيله في Kernel.php في $middlewareAliases.
  • ✅ تطبيقه على مسارات المشاريع في routes/web.php.
  • ✅ المستخدم غير المالك يحصل على 403 عند محاولة فتح مشروع غيره.
  • ✅ المستخدم المالك يمكنه الوصول لمشروعه بدون مشاكل.
  • ✅ [تحدي] إنشاء CheckRole Middleware مع parameter.

🚀 تحدي إضافي: سلسلة Middleware متعددة

طبق 3 Middleware على مسار واحد:
1. auth - يتأكد من تسجيل الدخول.
2. verified - يتأكد من توثيق البريد (يتطلب تفعيل MustVerifyEmail في User model).
3. check.project - يتأكد من صلاحية الوصول للمشروع.
