📝 كويز الدرس 15: Middleware

"اختبر فهمك لطبقات الحماية التي تفحص كل طلب قبل الوصول للـ Controller."

عدد الأسئلة: 9 (5 MCQ + 2 T/F + 1 نظري + 1 عملي)

1. ما هو Middleware في Laravel؟

✅ الشرح: Middleware هي طبقة وسيطة تفحص الطلب قبل وصوله للـ Controller. مثل البواب الإلكتروني.

2. ماذا يفعل Middleware 'auth' في Laravel؟

✅ الشرح: Middleware 'auth' يمنع الزوار غير المسجلين. إذا لم يسجل الدخول، يعيد توجيهه لصفحة login.

3. أين يتم تسجيل Middleware المخصص لاستخدامه في المسارات؟

✅ الشرح: نضيف الاسم المختصر (Alias) في $middlewareAliases في Kernel.php.

4. أي أمر ينشئ Middleware جديداً؟

✅ الشرح: php artisan make:middleware CheckProject ينشئ ملف Middleware جديد في app/Http/Middleware.

5. ماذا تعني $next($request) في دالة handle الخاصة بـ Middleware؟

✅ الشرح: $next($request) يمرر الطلب إلى الـ Middleware التالي في السلسلة أو إلى الـ Controller.

6. (صح/خطأ) Middleware 'guest' يمنع الزوار غير المسجلين من الوصول.

✅ الشرح: خطأ. 'guest' يفعل العكس - يسمح فقط للزوار غير المسجلين. المسجلين يعيد توجيههم للوحة التحكم.

7. (صح/خطأ) يمكن تطبيق أكثر من Middleware على نفس المسار.

✅ الشرح: صح. يمكن تطبيق عدة Middleware كمصفوفة: ->middleware(['auth', 'verified', 'check.project']).

8. (نظري) اشرح باختصار: لماذا نستخدم Middleware بدلاً من وضع نفس الشروط داخل كل Controller؟

💡 الإجابة النموذجية: الـ Middleware يطبق مبدأ DRY (Don't Repeat Yourself). بدلاً من كتابة نفس كود التحقق (مثل التأكد من تسجيل الدخول) في كل دالة Controller، نكتبه مرة واحدة في Middleware ونطبقه على كل المسارات التي تحتاجه. هذا يجعل الكود أنظف وأسهل للصيانة والتعديل.

9. (عملي) أكتب كود Middleware (دالة handle) يتحقق من أن المستخدم لديه دور "admin" ويسمح له فقط بالمرور. إذا لم يكن Admin، يرسل 403.

💡 الإجابة النموذجية:
public function handle(Request $request, Closure $next): Response
{
$user = $request->user();

if (!$user || $user->role !== 'admin') {
abort(403, 'يجب أن تكون مشرفاً للوصول لهذه الصفحة');
}

return $next($request);
}
