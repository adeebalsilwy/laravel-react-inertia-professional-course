صفحة الملف الشخصي وإعدادات الحساب

"يجب أن يشعر المستخدم بالسيطرة الكاملة على بياناته. صفحة الإعدادات هي مكانه لضبط حسابه."

1. لماذا نحتاج صفحة الملف الشخصي؟

أي تطبيق احترافي يجب أن يسمح للمستخدم بإدارة بياناته الشخصية. صفحة الملف الشخصي (Profile Page) هي المكان الذي يمكن للمستخدم من خلاله:

  • رؤية معلوماته (الاسم، البريد الإلكتروني، تاريخ التسجيل).
  • تغيير اسمه أو بريده الإلكتروني.
  • تغيير كلمة المرور الخاصة به.
  • رفع صورة شخصية (Avatar).
  • حذف حسابه بالكامل (اختياري).

في هذا الدرس، سنقوم بتخصيص صفحة الملف الشخصي التي يوفرها Laravel Breeze لتتناسب مع هوية FlowPilot، مع إضافة تحسينات مثل رفع الصور والرسائل الفورية.

2. عرض معلومات المستخدم

Laravel Breeze يوفر صفحة ملف شخصي جاهزة. يمكننا الوصول إليها عبر المسار /profile. المعلومات تظهر من خلال دالة edit() في ProfileController:

// app/Http/Controllers/ProfileController.php class ProfileController extends Controller { // عرض صفحة تعديل الملف الشخصي public function edit(Request $request) { // يمرر المستخدم الحالي إلى صفحة React return Inertia::render('Profile/Edit', [ 'user' => $request->user(), ]); } // تحديث معلومات المستخدم (الاسم والبريد) public function update(Request $request) { // ... (سنشرحها لاحقاً) } }

🔍 Inertia::render('Profile/Edit', ['user' => $request->user()]) يمرر بيانات المستخدم إلى صفحة React Profile/Edit.jsx.

// resources/js/Pages/Profile/Edit.jsx - عرض بيانات المستخدم import { usePage } from '@inertiajs/react'; export default function Edit() { const { props } = usePage(); const user = props.auth.user; // أو props.user return ( <div className="max-w-2xl mx-auto p-6"> <h1 className="text-3xl font-black">الملف الشخصي</h1> <div className="bg-white rounded-2xl p-6 shadow-sm mt-6"> <p><strong>الاسم:</strong> {user.name}</p> <p><strong>البريد:</strong> {user.email}</p> <p><strong>تاريخ التسجيل:</strong> {user.created_at}</p> </div> </div> ); }

🔍 user.name: يجلب اسم المستخدم. user.email: البريد الإلكتروني. user.created_at: تاريخ إنشاء الحساب.

3. نموذج تعديل الاسم والبريد الإلكتروني

نموذج التعديل يستخدم useForm من Inertia لإرسال البيانات بطريقة غير متزامنة (AJAX) مع الحفاظ على الـ CSRF Protection:

// resources/js/Pages/Profile/Edit.jsx - نموذج التعديل import { useForm } from '@inertiajs/react'; export default function Edit() { // useForm: يعطي دوال لإدارة النموذج (data, setData, post, put, errors, processing) const { data, setData, put, errors, processing } = useForm({ name: '', // القيمة الافتراضية email: '', }); // عند إرسال النموذج function submit(e) { e.preventDefault(); // منع إعادة تحميل الصفحة put('/profile'); // إرسال طلب PUT إلى /profile } return ( <form onSubmit={submit} className="space-y-4"> <div> <label className="block font-bold mb-1">الاسم</label> <input type="text" value={data.name} onChange={e => setData('name', e.target.value)} className="w-full p-3 border rounded-xl" /> {errors.name && <p className="text-red-500 text-sm mt-1">{errors.name}</p>} </div> <div> <label className="block font-bold mb-1">البريد الإلكتروني</label> <input type="email" value={data.email} onChange={e => setData('email', e.target.value)} className="w-full p-3 border rounded-xl" /> {errors.email && <p className="text-red-500 text-sm mt-1">{errors.email}</p>} </div> <button type="submit" disabled={processing} className="bg-indigo-600 text-white px-6 py-3 rounded-xl font-bold hover:bg-indigo-700" > {processing ? 'جاري الحفظ...' : 'حفظ التغييرات'} </button> </form> ); }

🔍 شرح الكود:

  • useForm({name: '', email: ''}): Hook من Inertia يدير حالة النموذج. data يحمل القيم الحالية، setData يحدثها، errors يحمل أخطاء التحقق من الخادم.
  • put('/profile'): يرسل طلب PUT (تحديث) إلى المسار /profile مع بيانات النموذج. Inertia يتعامل مع الـ CSRF تلقائياً.
  • e.preventDefault(): يمنع المتصفح من إعادة تحميل الصفحة (السلوك الافتراضي للنماذج).
  • errors.name && (

    ...): إذا وجد خطأ في حقل name من Laravel Validation، يعرض رسالة الخطأ.

  • disabled={processing}: يعطل الزر أثناء إرسال الطلب لمنع الضغط المتكرر.

4. نموذج تغيير كلمة المرور مع التحقق من كلمة المرور الحالية

لتغيير كلمة المرور، يجب على المستخدم إدخال كلمة المرور الحالية أولاً (كإجراء أمني). Laravel Breeze يوفر هذا النموذج جاهزاً:

// app/Http/Controllers/Auth/PasswordController.php public function update(Request $request) { // 1. التحقق من صحة البيانات (Validation) $request->validate([ // current_password: يجب أن يطابق كلمة المرور الحالية للمستخدم 'current_password' => ['required', 'current_password'], // password: مطلوب، 8 أحرف على الأقل، ويجب أن يطابق حقل confirmation 'password' => ['required', 'min:8', 'confirmed'], ]); // 2. تحديث كلمة المرور (Laravel يقوم بتشفيرها تلقائياً) $request->user()->update([ 'password' => bcrypt($request->password), ]); // 3. إعادة التوجيه مع رسالة نجاح return redirect()->back() ->with('message', 'تم تغيير كلمة المرور بنجاح'); }

🔍 شرح الكود:

  • 'current_password' => ['required', 'current_password']: قاعدة current_password هي قاعدة تحقق مدمجة في Laravel تتحقق من أن القيمة المدخلة تطابق كلمة مرور المستخدم الحالية.
  • 'password' => ['required', 'min:8', 'confirmed']: min:8 تتطلب 8 أحرف على الأقل. confirmed تتطلب وجود حقل password_confirmation وأن يطابق password.
  • bcrypt($request->password): تشفير كلمة المرور قبل تخزينها في قاعدة البيانات.
  • redirect()->back()->with('message', ...): إعادة التوجيه إلى الصفحة السابقة مع رسالة flash (مؤقتة).
// resources/js/Pages/Profile/Edit.jsx - نموذج تغيير كلمة المرور في React const passwordForm = useForm({ current_password: '', password: '', password_confirmation: '', }); function updatePassword(e) { e.preventDefault(); passwordForm.put('/password'); }

5. التحقق من صحة البيانات (Validation)

Validation (التحقق من الصحة) هو عملية التأكد من أن البيانات المدخلة تستوفي شروطاً معينة قبل حفظها. Laravel يوفر نظام Validation قوي وسهل.

// في ProfileController - التحقق من تحديث الاسم والبريد public function update(Request $request) { // 1. قواعد التحقق $request->validate([ 'name' => ['required', 'string', 'max:255'], // required: الحقل مطلوب // string: يجب أن يكون نصاً // max:255: الحد الأقصى 255 حرف 'email' => [ 'required', 'string', 'email', // يجب أن يكون بريداً إلكترونياً صحيحاً 'max:255', // unique: البريد يجب أن يكون فريداً في جدول users // ignore: يتجاهل بريد المستخدم الحالي (يسمح بنفس البريد) Rule::unique(User::class)->ignore($request->user()->id), ], ]); // 2. تحديث البيانات إذا نجح التحقق $request->user()->update([ 'name' => $request->name, 'email' => $request->email, ]); // 3. إعادة التوجيه مع رسالة نجاح return redirect()->back() ->with('message', 'تم تحديث الملف الشخصي بنجاح'); }

🔍 شرح قواعد التحقق:

  • required: الحقل لا يمكن أن يكون فارغاً.
  • string: يجب أن يكون الحقل من نوع نص.
  • email: يجب أن يكون بصيغة بريد إلكتروني صحيحة (name@domain.com).
  • max:255: الحد الأقصى لطول النص (255 حرف).
  • Rule::unique(User::class)->ignore(...): يتأكد من أن البريد غير مسجل مسبقاً، مع تجاهل بريد المستخدم الحالي (حتى لا يمنعه من تحديث بياناته دون تغيير البريد).

6. رسائل النجاح (Flash Messages)

Flash Messages (الرسائل المؤقتة) هي رسائل تظهر مرة واحدة للمستخدم بعد تنفيذ عملية (مثل "تم الحفظ بنجاح"). بعد تحديث الصفحة، تختفي الرسالة.

في React مع Inertia، نقرأ رسالة الفلاش من usePage().props.flash:

// resources/js/Pages/Profile/Edit.jsx - عرض رسالة النجاح const { props } = usePage(); const flash = props.flash; // { message: 'تم الحفظ بنجاح' } return ( <div> {flash.message && ( <div className="bg-green-100 border-r-4 border-green-500 text-green-700 p-4 rounded-xl mb-4"> {flash.message} </div> )} // ... باقي النموذج </div> ); }

🔍 flash.message: إذا كانت موجودة (غير null)، نعرضها في مربع أخضر. تحت الغطاء، HandleInertiaRequests هو المسؤول عن تمرير هذه الرسالة من Laravel Session إلى React Props.

🎯 مثال من الواقع: بطاقة الهوية الذكية

تخيل أن الملف الشخصي هو "بطاقة هوية" ذكية.

عرض المعلومات: مثل النظر إلى بطاقة هويتك - ترى اسمك وصورتك وتاريخ ميلادك.

تعديل الاسم: مثل الذهاب إلى مكتب الأحوال المدنية لتغيير اسمك - تحتاج لتقديم طلب (Request) مع البيانات الجديدة.

تغيير كلمة المرور: مثل تغيير رمز PIN لبطاقتك البنكية - يجب أن تعرف الرمز القديم أولاً لتتمكن من تعيين رمز جديد.

رسالة النجاح: مثل إيصال ورقي يقول "تم التحديث بنجاح". تمسكه بيدك ثم ترميه بعد قراءته (Flash Message - تظهر مرة واحدة فقط).

7. رفع الصورة الشخصية (Profile Photo)

إضافة صورة شخصية تجعل التطبيق أكثر احترافية. في Laravel، يمكننا رفع الصور باستخدام store() (تخزين الملفات):

// app/Http/Controllers/ProfileController.php - رفع الصورة public function updatePhoto(Request $request) { // 1. التحقق: الملف مطلوب، يجب أن يكون صورة، الحد الأقصى 2 ميجابايت $request->validate([ 'photo' => ['required', 'image', 'mimes:jpeg,png,jpg', 'max:2048'], ]); // 2. تخزين الصورة في مجلد 'profile-photos' داخل storage/app/public $path = $request->file('photo')->store('profile-photos', 'public'); // 3. حفظ المسار في قاعدة البيانات $request->user()->update([ 'profile_photo_path' => $path, ]); // 4. إنشاء رابط مباشر للصورة return redirect()->back() ->with('message', 'تم تحديث الصورة الشخصية'); }

🔍 شرح الكود:

  • 'image': يتأكد من أن الملف المرفوع هو صورة.
  • 'mimes:jpeg,png,jpg': يقبل فقط الصيغ: JPEG, PNG, JPG.
  • 'max:2048': الحد الأقصى لحجم الملف 2 ميجابايت (2048 كيلوبايت).
  • store('profile-photos', 'public'): يحفظ الملف في مجلد storage/app/public/profile-photos.
  • profile_photo_path: عمود في جدول users لحفظ مسار الصورة (يجب إضافته عبر Migration جديد).

🔗 إنشاء رابط للصورة في React:

<img src={`/storage/${user.profile_photo_path}`} alt="الصورة الشخصية" className="w-24 h-24 rounded-full object-cover" />

📌 تذكر: شغّل php artisan storage:link لإنشاء رابط رمزي (Symbolic Link) من /public/storage إلى /storage/app/public.

8. FlowPilot: صفحة إعدادات المستخدم

في FlowPilot، صفحة الإعدادات ستكون مقسمة إلى أقسام (Tabs):
القسم 1 - المعلومات الشخصية: تعديل الاسم والبريد.
القسم 2 - الأمان: تغيير كلمة المرور.
القسم 3 - الصورة: رفع صورة شخصية.
القسم 4 - حذف الحساب: زر لحذف الحساب بالكامل (مع تأكيد).

// resources/js/Pages/Profile/Edit.jsx - بنية صفحة الإعدادات في FlowPilot <div className="max-w-3xl mx-auto space-y-8"> // قسم المعلومات الشخصية <SectionCard title="المعلومات الشخصية"> <ProfileInfoForm /> </SectionCard> // قسم تغيير كلمة المرور <SectionCard title="تغيير كلمة المرور"> <UpdatePasswordForm /> </SectionCard> // قسم الصورة الشخصية <SectionCard title="الصورة الشخصية"> <ProfilePhotoForm /> </SectionCard> </div>

🔑 هذا التقسيم يجعل الصفحة سهلة الاستخدام ويسمح للمستخدم بالتركيز على قسم واحد في كل مرة.

9. التطبيق العملي: 7 خطوات

الخطوة 1: استعراض ملفات البروفايل الجاهزة

اذهب إلى المجلد resources/js/Pages/Profile/. ستجد ملفات جاهزة من Breeze: Edit.jsx (الصفحة الرئيسية) و Partials/ (مكونات فرعية).

الخطوة 2: تخصيص صفحة الملف الشخصي

عدّل Edit.jsx:

  • غيّر النصوص من الإنجليزية إلى العربية.
  • غيّر ألوان Tailwind لتناسب FlowPilot (indigo, pink).
  • أضف شعار FlowPilot في أعلى الصفحة.

الخطوة 3: تعديل دالة update في ProfileController

افتح app/Http/Controllers/ProfileController.php وتأكد من وجود التحقق (Validation) الصحيح للتحديث.

الخطوة 4: إضافة نموذج تغيير كلمة المرور

Breeze يوفر UpdatePasswordForm في resources/js/Pages/Profile/Partials/. تأكد من أنه مضمن في صفحة Edit.

الخطوة 5: إضافة رسائل النجاح Flash

أضف مكوناً صغيراً في أعلى صفحة البروفايل يعرض رسالة النجاح من usePage().props.flash.message.

الخطوة 6: [تحدي] إضافة رفع الصورة الشخصية

أنشئ مكون ProfilePhotoForm في React يسمح برفع صورة. أضف مساراً في Controller لاستقبال الصورة وحفظها.

الخطوة 7: [تحدي] إضافة رسالة تأكيد قبل الحفظ

عند تغيير البريد الإلكتروني، أضف رسالة تأكيد (Confirm Dialog) تسأل المستخدم "هل أنت متأكد من تغيير البريد؟".

11. الأخطاء الشائعة

❌ الخطأ 1: "The email field is required." يظهر رغم ملء الحقل

السبب: اسم الحقل في Form React لا يطابق اسم الحقل المتوقع في Laravel.

الحل: تأكد أن اسم الحقل في setData('email', ...) هو نفسه اسم الحقل في $request->validate([...]).

❌ الخطأ 2: "The password field confirmation does not match."

السبب: حقل password_confirmation لا يطابق password.

الحل: تأكد من وجود حقل باسم password_confirmation في النموذج React.

❌ الخطأ 3: Flash message لا يظهر في React

السبب: HandleInertiaRequests.php لا يشارك flash messages.

الحل: تأكد من وجود مفتاح 'flash' في دالة share() داخل HandleInertiaRequests.

❌ الخطأ 4: الصورة لا تظهر بعد الرفع

السبب: لم يتم إنشاء الرابط الرمزي (Symbolic Link) للتخزين.

الحل: شغّل php artisan storage:link.
