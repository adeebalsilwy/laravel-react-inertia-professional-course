📝 كويز الدرس 16: الملف الشخصي والإعدادات

"اختبر فهمك لكيفية بناء صفحة إعدادات متكاملة وآمنة للمستخدم."

عدد الأسئلة: 9 (5 MCQ + 2 T/F + 1 نظري + 1 عملي)

1. أي Hook من Inertia يستخدم لإدارة نموذج (Form) في React؟

✅ الشرح: useForm هو Hook خاص بـ Inertia لإدارة النماذج مع دوال مثل put, post, errors, processing.

2. أي طلب HTTP يستخدم لتحديث بيانات المستخدم في ProfileController؟

✅ الشرح: طلب PUT يستخدم لتحديث (Update) الموارد. useForm().put('/profile') يرسل طلب PUT.

3. ما قاعدة التحقق (Validation Rule) التي تتأكد من أن البريد الإلكتروني غير مكرر في قاعدة البيانات؟

✅ الشرح: Rule::unique(User::class) يتأكد من أن القيمة غير موجودة في جدول users. مع ->ignore($id) يتجاهل المستخدم الحالي.

4. كيف نصل إلى بيانات المستخدم المسجل في صفحة React الخاصة بالبروفايل؟

✅ الشرح: Inertia يشارك بيانات المستخدم عبر HandleInertiaRequests، ويمكن الوصول إليها عبر usePage().props.auth.user.

5. ماذا يفعل e.preventDefault() في دالة submit للنموذج؟

✅ الشرح: e.preventDefault() يمنع المتصفح من إعادة تحميل الصفحة عند إرسال النموذج، مما يسمح لـ Inertia بالتعامل مع الطلب.

6. (صح/خطأ) لتغيير كلمة المرور، يجب على المستخدم إدخال كلمة المرور الحالية أولاً.

✅ الشرح: صح. قاعدة current_password تتحقق من أن القيمة تطابق كلمة المرور الحالية للمستخدم. هذا إجراء أمني.

7. (صح/خطأ) رسائل Flash Messages تبقى ظاهرة لعدة زيارات للصفحة حتى يغلقها المستخدم يدوياً.

✅ الشرح: خطأ. Flash Messages تظهر مرة واحدة فقط (عند إعادة التوجيه التالية). بعد تحديث الصفحة تختفي تلقائياً.

8. (نظري) لماذا نستخدم قاعدة Rule::unique(User::class)->ignore($user->id) عند التحقق من البريد الإلكتروني في ProfileController؟

💡 الإجابة النموذجية: قاعدة unique تمنع تسجيل بريد إلكتروني مكرر. لكن عند تحديث الملف الشخصي، المستخدم قد لا يغير بريده، مما قد يسبب خطأ "البريد موجود مسبقاً" لأن بريده موجود بالفعل في قاعدة البيانات. ignore($user->id) يقول: "تجاهل السجل الخاص بهذا المستخدم - اسمح بنفس البريد للمالك الحالي".

9. (عملي) أكتب كود React (باستخدام useForm) لنموذج يرسل طلب PUT إلى /profile لتحديث الاسم فقط. أظهر رسالة خطأ إذا فشل التحقق.

💡 الإجابة النموذجية:
import { useForm } from '@inertiajs/react';

export default function UpdateName() {
const { data, setData, put, errors, processing } = useForm({ name: '' });

function submit(e) {
e.preventDefault();
put('/profile');
}

return (
<form onSubmit={submit}>
<input value={data.name} onChange={e => setData('name', e.target.value)} />
{errors.name && <p>{errors.name}</p>}
<button disabled={processing}>حفظ</button>
</form>
);
}
