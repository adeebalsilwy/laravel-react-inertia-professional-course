اختبر فهمك: Git وحفظ التقدم

اختبر معلوماتك عن نظام التحكم في الإصدارات.

1. ما هو الأمر الذي تستخدمه لإنشاء مستودع Git جديد في مجلد مشروعك؟

الشرح: git init هو الأمر الذي ينشئ مجلد .git/ في المشروع، محولاً إياه إلى مستودع Git. "init" اختصار لـ Initialize (تهيئة).

2. ما هي وظيفة Staging Area في Git؟

الشرح: Staging Area تسمح لك بتحديد أي الملفات تريد حفظها في الـ Commit. يمكنك إضافة بعض الملفات git add file1.js وتجاهل البعض الآخر. هذا يعطيك تحكماً دقيقاً في كل Commit.

3. ماذا يفعل الأمر git add . (بنقطة)؟

الشرح: النقطة . في git add . تشير إلى "المجلد الحالي". الأمر يضيف كل الملفات في المجلد الحالي والمجلدات الفرعية (عدا المستثناة في .gitignore) إلى Staging Area.

4. ما هو الغرض من ملف .gitignore؟

الشرح: .gitignore يحدد الملفات والمجلدات التي يجب على Git تجاهلها. في Laravel، نتجاهل /vendor (لأن Composer يعيد إنشائه)، /node_modules (لأن NPM يعيد إنشائه)، و .env (لأنه يحتوي كلمات سر).

5. أي أمر يعرض تاريخ الـ Commits في المستودع؟

الشرح: git log يعرض سجل الـ Commits مرتباً من الأحدث للأقدم. يظهر لكل Commit: Hash، Author، Date، والرسالة. يمكنك استخدامه لرؤية تاريخ المشروع بالكامل.

6. صح أم خطأ: بعد تنفيذ git commit، يمكنك العودة لحالة المشروع قبل الـ Commit إذا أخطأت.

الشرح: صح. هذا هو جوهر Git. كل Commit هو نقطة حفظ يمكنك العودة إليها بأمر git checkout <hash>. يمكنك أيضاً استخدام git revert للتراجع عن Commit معين دون فقدان التاريخ.

7. صح أم خطأ: ملف vendor/ يجب أن يكون ضمن Commits Git لأنه ضروري للمشروع.

الشرح: خطأ. مجلد vendor/ يتم إنشاؤه بواسطة Composer بأمر composer install. لا نحتاج لتتبعه في Git لأن أي شخص يستنسخ المشروع يمكنه تشغيل composer install لإنشائه. تتبعه سيجعل المستودع ضخماً جداً (آلاف الملفات).

8. سؤال نظري: اشرح بجملتين الفرق بين git status و git log.

مثال للإجابة: git status يعرض الحالة الحالية للمشروع: الملفات المعدلة، الملفات الجديدة، والملفات في Staging Area. git log يعرض تاريخ المشروع: جميع الـ Commits السابقة مع تفاصيلها. الأول للحاضر، والثاني للماضي.

9. سؤال تطبيقي: اكتب الأوامر الثلاثة المطلوبة لعمل أول Commit لمشروعك (بفرض أن git init قد تم بالفعل).

الإجابة الصحيحة:
1. git add . - إضافة جميع الملفات إلى Staging
2. git commit -m "رسالتك" - حفظ الـ Commit
3. git status أو git log - التحقق من النتيجة
