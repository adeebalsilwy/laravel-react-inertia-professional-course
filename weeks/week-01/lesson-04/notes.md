# ملخص الدرس: Git وتنظيم المستودع والمعايير

## الفكرة الأساسية
Git هو نظام حفظ إصدارات. يسمح لنا بمعرفة ماذا تغير في المشروع، ومن قام بالتغيير، والعودة إلى نسخة سابقة عند الحاجة. GitHub أو GitLab يمكن أن يستضيف المستودع ويسهل مشاركة المشروع.

## لماذا هذا مهم؟
دورة مدتها شهرين ستضيف عشرات الملفات والتعديلات. بدون Git سيصعب تتبع الأخطاء، وسيخاف الطالب من التجربة لأن الرجوع للخلف غير واضح.

## داخل FlowPilot
كل درس في FlowPilot يمكن أن يكون commit مستقل. هذا يجعل المشروع التعليمي منظم ويمكن مراجعته لاحقاً خطوة بخطوة.

## المثال
git init
git add .
git commit -m "Create initial FlowPilot structure"
git status

## نتيجة الدرس
مستودع منظم يحتوي على أول commit وملف README واضح.
