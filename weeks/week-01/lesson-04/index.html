Git وتنظيم المستودع والمعايير البرمجية Clean Code

"المبرمج المحترف ليس من يكتب كوداً يعمل فحسب، بل من يكتب كوداً منظماً يستطيع الجميع قراءته."

ما هو نظام Git؟

تخيل أنك تكتب بحثاً طويلاً، وفي كل مرة تقوم بتعديله تحفظه باسم جديد: "بحث_نهائي"، "بحث_نهائي_حقاً"، "بحث_نهائي_أخير". هذا أسلوب فوضوي!

Git هو نظام ذكي يقوم بـ "تصوير" مشروعك في كل مرحلة مهمة. إذا أخطأت لاحقاً، يمكنك ببساطة إخبار Git: "أرجوك عد بي إلى نسخة يوم الثلاثاء الماضي قبل أن أفسد قاعدة البيانات".

ما هو الكود النظيف (Clean Code)؟

في FlowPilot، لن نكتب أسماء متغيرات عشوائية مثل $x أو $data1. سنتبع معايير عالمية:

  • تسمية المتغيرات بأسماء واضحة (مثلاً: $activeProjects).
  • كتابة تعليقات تشرح "لماذا" فعلنا هذا، وليس "ماذا" يفعل الكود.
  • تنظيم الملفات في مجلداتها الصحيحة.

أول "لقطة" لمشروعك (First Commit)

1

تفعيل Git في المشروع

git init

git add .

git commit -m "Initial: تثبيت مشروع FlowPilot وتهيئة البيئة"

شرح الأوامر: init (بدأ النظام)، add (تجهيز الملفات)، commit (حفظ اللقطة مع رسالة وصفية).
