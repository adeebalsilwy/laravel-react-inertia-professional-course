Git: آلة الزمن والتعاون البرمجي في FlowPilot

"المبرمج المحترف لا يخاف من الخطأ، لأنه يمتلك القدرة على العودة بالزمن وإصلاحه في ثوانٍ."

لماذا هذا الدرس مهم؟

هل سبق لك وأن قمت بتعديل ملف ثم ندمت وأردت العودة للشكل القديم؟ أو هل خفت من تجربة كود جديد يفسد مشروعك بالكامل؟ هنا تأتي قوة Git.

في مشروع FlowPilot، سنستخدم Git لحفظ "لقطات" (Commits) من تقدمنا. هذا يضمن لنا أن تعبنا لن يضيع أبداً، ويسمح لنا بالتعاون مع مبرمجين آخرين بسهولة واحترافية.

ما هو Git؟

Git هو نظام "تتبع الإصدارات" (Version Control System). وببساطة هو برنامج يراقب كل حرف تكتبه في مشروعك.

  • Repository (المستودع): هو المجلد الذي يحتوي على مشروعك وتاريخه بالكامل.
  • Commit (الحفظ): هي عملية أخذ "لقطة" لكودك في لحظة معينة مع رسالة تشرح ما فعلته.
  • Push (الرفع): هو إرسال مشروعك من جهازك إلى الإنترنت (مثل GitHub) ليكون آمناً.

مثال من الواقع (حفظ الألعاب)

تخيل أنك تلعب لعبة صعبة. قبل أن تقابل "الوحش"، ستقوم بعمل Save للعبتك. إذا خسرت، لن تبدأ اللعبة من الصفر، بل ستعود لنقطة الحفظ الأخيرة وتجرب مرة أخرى بذكاء أكبر.

Git هو تماماً مثل زر Save في الألعاب، والـ Commit هي نقطة الحفظ التي تعود إليها إذا أفسدت الكود!

التطبيق العملي

الخطوة 1: بدء تشغيل Git في FlowPilot

git init

هذا الأمر ينشئ "المستودع" داخل مجلد مشروعك.

الخطوة 2: حفظ أول تقدم (Commit)

git add .

git commit -m "إتمام تهيئة مشروع FlowPilot وربطه بـ React"
