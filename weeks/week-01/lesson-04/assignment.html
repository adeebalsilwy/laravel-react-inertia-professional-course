واجب الدرس 4: أول Commit في تاريخ FlowPilot

"هذه هي اللحظة التي يصبح فيها مشروعك رسمياً تحت الحماية. أول Commit هو شهادة ميلاد مشروعك الرقمي."

01 ضبط هوية Git وتفعيل المستودع

الهدف:

تعريف Git بهويتك وإنشاء أول مستودع.

الخطوات:

  1. 1. افتح Terminal داخل مجلد flowpilot.
  2. 2. git config --global user.name "اسمك الكامل"
  3. 3. git config --global user.email "بريدك@example.com"
  4. 4. git init

تأكد من ظهور رسالة "Initialized empty Git repository" بدون أخطاء.

ماذا تتحقق منه؟

  • ظهور رسالة التأكيد من git init.
  • (اختياري) في File Explorer، فعّل "Show hidden files" لترى مجلد .git.

02 التحقق من .gitignore والملفات المتتبعة

الهدف:

التأكد من أن ملفات vendor و node_modules مستثناة من التتبع.

  1. 1. تحقق من وجود ملف .gitignore في جذر المشروع: Get-Content .gitignore
  2. 2. تأكد من احتوائه على /vendor و /node_modules و .env.
  3. 3. إذا لم يكن موجوداً، أنشئه واكتب فيه هذه الأسطر.

03 أول Commit رسمي

الهدف:

حفظ جميع ملفات المشروع في أول Commit.

الخطوات:

  1. 1. git add . (أضف كل الملفات إلى Staging)
  2. 2. git commit -m "تأسيس مشروع FlowPilot مع Laravel 12 و React و Inertia"
  3. 3. git status (يجب أن يظهر "nothing to commit")
  4. 4. git log (يجب أن يظهر Commit واحد)

💡 التقط لقطة شاشة لنتيجة git log - هذا دليل إنجازك!

★ التحدي الإضافي: إنشاء فرع (Branch)

للمتفوقين - جرب مفهوم الفروع:

  1. أنشئ فرعاً جديداً: git branch feature-welcome
  2. انتقل إليه: git checkout feature-welcome
  3. عد للفرع الرئيسي: git checkout main
  4. شاهد قائمة الفروع: git branch (الفرع النشط عليه علامة *)

التقط صورة لنتيجة git branch تظهر فرعيْن.

04 أسئلة فهم

  1. ما الفرق بين git add و git commit؟
  2. ما هي وظيفة ملف .gitignore؟ ولماذا نتجاهل vendor/؟
  3. لماذا نكتب رسالة Commit واضحة؟ أعط مثالاً على رسالة جيدة وأخرى سيئة.
  4. اشرح بأسلوبك: لماذا يعتبر Git "آلة الزمن" للمبرمج؟

طريقة التسليم

  • • لقطة شاشة 1: نتيجة git log تظهر Commit الأول.
  • • لقطة شاشة 2: نتيجة git status تظهر "nothing to commit".
  • • لقطة شاشة 3 (اختياري): نتيجة git branch مع فرعيْن.
  • • النص: إجابات الأسئلة النظرية.
  • ⚠️ تهانينا! بإتمام هذا الواجب، تكون قد أتممت الأسبوع الأول بالكامل. أنت الآن تمتلك أساساً متيناً لمشروع FlowPilot!

مخاطر شائعة في Git

  • • نسيان .gitignore: سيحاول Git تتبع مجلد vendor/ (آلاف الملفات) وقد يتلف المستودع.
  • • عدم ضبط user.name و user.email: سيظهر Commit باسم مجهول.
  • • رسائل Commit غير مفيدة: تجنب "fix" أو "update". اكتب وصفاً دقيقاً.
  • • كتابة كلمة سر في ملف عادي: Git سيتذكرها للأبد. استخدم .env دائماً.
