# واجب الدرس 1: تهيئة بيئة العمل وفهم خريطة مشروع FlowPilot

## الهدف من الواجب
تطبيق عملي لتثبيت والتحقق من جميع الأدوات المطلوبة لبدء مشروع FlowPilot. بعد هذا الواجب، سيكون جهازك جاهزاً تماماً لإنشاء أول مشروع Laravel.

## المطلوب بالتفصيل

### الجزء 1: فحص الأدوات (إجباري)
1. افتح Terminal (cmd أو PowerShell) على ويندوز.
2. نفذ الأوامر التالية وصوّر النتيجة:
   - `php -v` (يجب أن يظهر إصدار 8.2+)
   - `composer -v` (يجب أن يظهر إصدار Composer)
   - `node -v` (يجب أن يظهر إصدار 18+)
   - `npm -v` (يجب أن يظهر إصدار NPM)
   - `git --version` (يجب أن يظهر إصدار Git)
3. التقط لقطة شاشة واحدة تظهر جميع النتائج.

### الجزء 2: تجهيز VS Code (إجباري)
1. ثبت VS Code من code.visualstudio.com.
2. افتح VS Code.
3. اذهب لإدارة الإضافات (Ctrl+Shift+X).
4. ابحث عن هذه الإضافات وثبتها:
   - PHP Intelephense
   - Laravel Extension Pack
   - ES7+ React/Redux Snippets
   - Tailwind CSS IntelliSense
   - GitLens
5. التقط صورة لـ VS Code مع الإضافات الظاهرة.

### الجزء 3: التحدي الإضافي (اختياري - للمتفوقين)
1. أنشئ مجلد `FlowPilot-Academy` على سطح المكتب.
2. افتحه في VS Code (File → Open Folder).
3. أنشئ ملف `env-check.txt`.
4. اكتب فيه تاريخ اليوم ونتائج الفحص.
5. احفظ الملف.

## معيار القبول
- ظهور جميع الأوامر بنجاح في Terminal دون أخطاء.
- VS Code مثبت مع الإضافات الخمسة.
- شرح نظري صحيح لدور كل أداة.

## طريقة التسليم
- الصور: صورتان (واحدة للـ Terminal، واحدة لـ VS Code).
- ملف النص: يحتوي على الشرح.
- التحدي الإضافي: ملف `env-check.txt`.

## مخاطر شائعة
- لا تنس إعادة فتح Terminal بعد تثبيت أي أداة جديدة.
- تأكد من تنزيل PHP 8.2+ وليس 7.x.
- استخدم الروابط الرسمية فقط، لا تثق بمواقع التحميل غير المعروفة.
- إذا ظهر خطأ "is not recognized"، هذا يعني أن البرنامج غير مضاف إلى PATH.
