واجب الدرس الأول: تجهيز صندوق أدوات FlowPilot

"الواجب ليس مجرد خطوات تقنية - إنه أول إنجاز حقيقي لك في رحلة البرمجة. بعد إتمامه، سيكون جهازك جاهزاً لبناء أي مشروع ويب."

01 فحص الأدوات في Terminal (الجزء الأساسي)

الهدف: التأكد من أن جميع الأدوات مثبتة وتعمل بشكل صحيح.

الخطوات المطلوبة بالتفصيل:

  1. 1. افتح Terminal (cmd أو PowerShell).
  2. 2. اكتب الأمر php -v واضغط Enter.
  3. 3. اكتب الأمر composer -v واضغط Enter.
  4. 4. اكتب الأمر node -v واضغط Enter.
  5. 5. اكتب الأمر npm -v واضغط Enter.
  6. 6. اكتب الأمر git --version واضغط Enter.

ماذا يجب أن ترى بالضبط؟

لكل أمر، يجب أن يظهر رقم إصدار وليس رسالة خطأ. مثلاً:

  • PHP 8.2.26 (cli) (built: ... )
  • Composer version 2.8.3
  • v22.11.0
  • 10.9.0
  • git version 2.47.0.windows.2

طريقة التسليم:

التقط لقطة شاشة (Screenshot) كاملة للنافذة بعد كتابة جميع الأوامر. يجب أن تظهر جميع النتائج في صورة واحدة.

02 تثبيت VS Code والإضافات

الهدف: تثبيت محرر الكود الأساسي وتجهيزه لتطوير FlowPilot.

  1. 1. اذهب إلى code.visualstudio.com وحمل VS Code.
  2. 2. ثبت البرنامج (اختر "Add to PATH" أثناء التثبيت).
  3. 3. افتح VS Code، اذهب لإدارة الإضافات (Ctrl+Shift+X).
  4. 4. ابحث عن الإضافات التالية وقم بتثبيتها:
    • PHP Intelephense (Ben Mewburn)
    • Laravel Extension Pack (Winnie Lin)
    • ES7+ React/Redux Snippets
    • Tailwind CSS IntelliSense
    • GitLens

طريقة التسليم: التقط صورة لـ VS Code مفتوحاً مع ظهور الإضافات المثبتة في الشريط الجانبي.

03 تحدي الشرح: اشرح لأخيك المبتدئ

المطلوب منك (تحدي فهم، لا يحتاج Terminal):

اكتب فقرة (5-10 أسطر) تشرح فيها لشخص لا يعرف أي شيء عن البرمجة لماذا نحتاج كل أداة. استخدم التشبيهات. مثلاً:

"PHP هو الطباخ، Composer هو من يشتري المكونات، Node.js هو الفرن، Git هو كاميرا المراقبة..."

معايير التقييم:

  • الوضوح: هل يستطيع شخص عادي فهمك؟
  • الدقة: هل ذكرت الوظيفة الصحيحة لكل أداة؟
  • الإبداع: هل استخدمت تشبيهاً مبتكراً؟

★ التحدي الإضافي (اختياري)

للمتقدمين فقط - هذا الجزء غير مطلوب للنجاح لكنه سيمنحك خبرة إضافية:

  1. قم بإنشاء مجلد باسم FlowPilot-Academy على سطح المكتب.
  2. افتح هذا المجلد في VS Code (File → Open Folder).
  3. أنشئ ملف جديد باسم env-check.txt.
  4. انسخ فيه نتائج الأوامر الستة من الجزء الأول.
  5. احفظ الملف باستخدام Ctrl+S.

هذا التحدي يعلمك التعامل مع الملفات في VS Code - مهارة أساسية.

طريقة التسليم النهائية

  • • الجزء 1 + 2: صورتان (واحدة للـ Terminal، واحدة لـ VS Code مع الإضافات).
  • • الجزء 3: نص الشرح (يفضل أن يكون في ملف Word أو نصي).
  • • الجزء 4 (اختياري): ملف env-check.txt.
  • ⚠️ تذكير مهم جداً: إذا لم تتمكن من إكمال هذا الواجب، راجع الدرس مرة أخرى. الدروس القادمة تعتمد على أن أدواتك جاهزة.

مخاطر ومشاكل شائعة يجب الانتباه لها

  • • نسخ الأوامر من الإنترنت: قد تحتوي الأوامر المنقولة من مواقع غير موثوقة على مسافات مخفية أو أحرف خاصة. اكتب الأوامر بنفسك.
  • • نسخة PHP غير صحيحة: تأكد أنك تثبت PHP 8.2+ وليس 7.x. Laravel 12 لا يعمل على الإصدارات القديمة.
  • • عدم إغلاق وإعادة فتح Terminal: بعد تثبيت أداة جديدة، أعد فتح الـ Terminal حتى يتعرف على المسار الجديد.
  • • تجاهل رسائل التحذير: إذا ظهرت رسالة تحذير صفراء أو حمراء، لا تتجاهلها. اقرأها، ابحث عن معناها، ثم قرر.
  • • تثبيت برامج من مصادر غير رسمية: استخدم فقط الموقع الرسمي لكل أداة.
