اختبر فهمك: بيئة العمل والأدوات

أجب عن الأسئلة التالية. بعد الانتهاء، اضغط "تصحيح الإجابات" لترى نتائجك.

1. ما هي الوظيفة الأساسية لبرنامج Composer في مشروع FlowPilot؟

الشرح: Composer هو مدير حزم PHP. وظيفته الأساسية هي تحميل وتحديث وإدارة المكتبات الجاهزة التي يحتاجها مشروع Laravel. مثلاً، هو من قام بتحميل Laravel نفسه عندما كتبنا أمر create-project.

2. لماذا نحتاج إلى Node.js في مشروع Laravel؟

الشرح: PHP و Laravel يعملان على الخادم لتجهيز البيانات. أما الواجهة التي يراها المستفظ فهي React (JavaScript). Node.js هو المحرك الذي يسمح لنا بمعالجة كود React و Tailwind CSS وتحويلهما إلى شيء يعمل في المتصفح.

3. أي من الإصدارات التالية يعمل مع Laravel 12؟

الشرح: Laravel 12 يتطلب PHP 8.2 أو أحدث. الإصدارات الأقدم لا تدعم الميزات الحديثة التي يحتاجها Laravel، وقد تسبب أخطاء غير متوقعة.

4. ما وظيفة Git في تطوير البرمجيات؟

الشرح: Git هو نظام التحكم في الإصدارات. يسجل كل تغيير تجريه على الكود ويمكنك من العودة لأي نقطة سابقة. مثلاً، إذا أضفت ميزة جديدة وكسرت شيئاً قديماً، يمكنك العودة فوراً للحظة قبل إضافة الميزة.

5. ماذا يعني المصطلح "تهيئة بيئة العمل" (Environment Setup)؟

الشرح: تهيئة البيئة تعني تثبيت وتجهيز جميع الأدوات (PHP، Composer، Node، Git، VS Code) التي سنستخدمها لاحقاً لتطوير تطبيق FlowPilot. إنها تشبه تجهيز أدوات النجارة قبل البدء في صنع الطاولة.

6. صح أم خطأ: يمكننا استخدام Composer لإدارة مكتبات JavaScript؟

الشرح: الإجابة هي "خطأ". Composer مخصص لحزم PHP فقط. بينما NPM هو المسؤول عن إدارة حزم JavaScript. كل أداة تخدم لغتها الخاصة.

7. صح أم خطأ: Git هو نفسه GitHub، وكلاهما يعملان بنفس الطريقة.

الشرح: الإجابة "خطأ". Git هو برنامج مثبت على جهازك لإدارة الإصدارات محلياً. GitHub هو موقع ويب يستضيف مشاريع Git على الإنترنت لتسهيل التعاون والمشاركة. الفرق مثل: حاسوبك الشخصي (Git) مقابل Dropbox (GitHub).

8. سؤال نظري: لماذا نشبه PHP بالطباخ و Composer بالمساعد في تشبيه المطبخ؟

مثال للإجابة الصحيحة: PHP هو الطباخ لأنه ينفذ الأوامر ويجهز البيانات (مثل الطباخ الذي يطبخ الطعام). Composer هو المساعد الذي يذهب للسوق لشراء المكونات الجاهزة (المكتبات) التي يحتاجها الطباخ. الطباخ لا يستطيع العمل بدون المكونات، و Composer لا يستطيع الطهي بنفسه - كل مكمل للآخر.

9. سؤال تطبيقي: اكتب الأمر الصحيح (مع الشرح) الذي تستخدمه في Terminal للتأكد من إصدار PHP المثبت على جهازك.

الإجابة الصحيحة: الأمر هو "php -v". php = اسم البرنامج، -v = اختصار لـ version (يطلب من PHP عرض إصدارها). أو يمكن أيضاً استخدام "php --version" بنفس المعنى. عند تنفيذه، سيظهر إصدار PHP المثبت (يجب أن يكون 8.2+).
