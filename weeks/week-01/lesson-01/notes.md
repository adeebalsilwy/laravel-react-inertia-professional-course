# ملاحظات الدرس 1: تهيئة بيئة العمل وفهم خريطة مشروع FlowPilot

## المفاهيم الأساسية
- **بيئة العمل (Environment):** مجموعة البرامج والأدوات المثبتة على جهاز الكمبيوتر والتي تسمح للمطور بكتابة وتشغيل التطبيقات.
- **PHP 8.2+:** لغة برمجة الخلفية. يجب أن لا يقل الإصدار عن 8.2.
- **Composer:** مدير حزم PHP. يتصل بـ Packagist.org لتحميل المكتبات.
- **Node.js 18+:** بيئة تشغيل JavaScript خارج المتصفح. مطلوب لتشغيل أدوات الواجهة.
- **NPM:** مدير حزم JavaScript. يتصل بـ npmjs.com.
- **Git:** نظام التحكم في الإصدارات. يسجل تاريخ التغييرات.
- **VS Code:** محرر نصوص ذكي مع إضافات للتطوير.
- **Terminal:** واجهة أوامر نصية للتفاعل مع النظام.
- **PATH:** متغير نظام يخبر Windows أين توجد البرامج القابلة للتنفيذ.

## أين يظهر في FlowPilot؟
- كل أداة من هذه الأدوات ستستخدم طوال الدورة.
- PHP و Composer هما قلب مشروع Laravel.
- Node.js و NPM هما قلب واجهة React و Tailwind.
- Git هو شبكة الأمان لحفظ كل تقدم.

## روابط التحميل الرسمية
- PHP: https://windows.php.net/download (أو استخدم Laragon/Herd)
- Composer: https://getcomposer.org
- Node.js: https://nodejs.org (اختر LTS)
- Git: https://git-scm.com
- VS Code: https://code.visualstudio.com

## أخطاء شائعة يجب تذكرها
- "is not recognized" ← لم تضف البرنامج إلى PATH
- إصدار قديم ← أعد التثبيت
- تثبيت برامج من مصادر غير رسمية ← استخدم فقط الروابط أعلاه
- نسيان إعادة فتح Terminal بعد التثبيت ← أعد الفتح دائماً
