تثبيت Laravel 12 وربطه بـ React عبر جسر Inertia.js

"اليوم سنبدأ فعلياً في بناء الأساس البرمجي لـ FlowPilot. سنجمع بين قوة Laravel وجمال React."

لماذا هذا الدرس مهم؟

في السابق، كان بناء تطبيق ويب يتطلب إنشاء مشروعيْن منفصلين؛ واحد للبيانات (Back-end) وآخر للشكل (Front-end). لكن مع تقنية Inertia.js، سنقوم بدمجهما في مشروع واحد ذكي.

هذا الدرس سيعلمك كيف تنشئ مجلد المشروع الحقيقي، وكيف تجعل Laravel و React يتواصلان معاً بسلاسة. بعد هذا الدرس، ستكون قادراً على رؤية أول صفحة تفاعلية لـ FlowPilot.

ما هو الـ Stack التقني؟

الـ Stack هو مجموعة التقنيات التي تعمل معاً لبناء التطبيق. في FlowPilot نستخدم:

  • Laravel 12: مسؤول عن "العقل"؛ حفظ البيانات، الصلاحيات، والتحقق من المستخدمين.
  • React: مسؤول عن "الواجهة"؛ الأزرار، القوائم، والتفاعل السريع دون إعادة تحميل الصفحة.
  • Inertia.js: هو "الجسر"؛ ينقل البيانات من Laravel إلى React بطريقة تجعل التطبيق يبدو كتطبيقات الموبايل السريعة.

مثال لتقريب الفكرة (المطعم الذكي)

تخيل Laravel هو المطبخ (حيث تُجهز الأطباق)، و React هو الصالة (حيث يجلس الزبائن).

بدون Inertia، سيكون على الزبون الذهاب للمطبخ في كل مرة يطلب فيها شيئاً جديداً.

مع Inertia، هناك نادل فائق السرعة يأخذ الطلب من الصالة، يذهب للمطبخ، ويعود بالطبق ويضعه على الطاولة دون أن يشعر الزبون بأي تأخير أو يحتاج لتغيير طاولته!

في مشروع FlowPilot

هذا هو اليوم الذي ننشئ فيه مجلد flowpilot الحقيقي على جهازك. كل الدروس القادمة ستعتمد على ما سنفعله اليوم. سننشئ "قلب" المشروع الذي سنبني عليه كل ميزات إدارة المهام والفرق لاحقاً.

التطبيق العملي خطوة بخطوة

الخطوة 1: إنشاء مشروع Laravel

الهدف: إنشاء المجلد الأساسي للمشروع وتحميل ملفات Laravel 12.

لماذا؟ نحتاج لنسخة نظيفة من Laravel لنبدأ بناء FlowPilot عليها.

composer create-project laravel/laravel flowpilot

cd flowpilot

شرح الأمر: نطلب من Composer إنشاء مشروع جديد اسمه flowpilot ثم ندخل إلى المجلد.

الخطوة 2: تثبيت Inertia جانب السيرفر

الهدف: إضافة مكتبة Inertia داخل ملفات Laravel.

composer require inertiajs/inertia-laravel

الخطوة 3: تثبيت React و Inertia جانب الواجهة

الهدف: تحميل مكتبات React لتشغيل الواجهات.

npm install @inertiajs/react react react-dom @vitejs/plugin-react

شرح كود الربط الأساسي

return Inertia::render('Home');

هذا السطر يخبر Laravel: "لا تبحث عن صفحة HTML عادية، بل ابحث عن صفحة React اسمها Home في مجلد الصفحات، وأرسلها للمستخدم عبر جسر Inertia".

أخطاء قد تواجهك

خطأ: Composer is not recognized

يعني أنك لم تثبت Composer أو لم تضفه للمسار.

الحل: عد للدرس الأول وتأكد من تثبيت Composer بنجاح.

ملخص الدرس

اليوم قمنا بإنشاء مشروع FlowPilot وربطنا Laravel بـ React باستخدام Inertia. هذه الخطوة هي التي حولت مشروعنا من مجرد فكرة إلى تطبيق ويب حديث جاهز للتطوير.
