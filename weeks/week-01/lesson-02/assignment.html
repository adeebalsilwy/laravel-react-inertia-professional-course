واجب الدرس 2: إنشاء أول مشروع FlowPilot

"هذه هي اللحظة التي يولد فيها مشروعك. سنقوم بإنشاء Laravel، ربطه بـ React، وتشغيل أول صفحة."

01 إنشاء مشروع Laravel

الهدف:

استخدام Composer لإنشاء مشروع Laravel 12 باسم flowpilot.

الخطوات:

  1. 1. افتح Terminal في أي مكان تريد (مثلاً سطح المكتب).
  2. 2. اكتب: composer create-project laravel/laravel flowpilot
  3. 3. انتظر حتى يكتمل التحميل (قد يستغرق 1-3 دقائق).
  4. 4. اكتب: cd flowpilot لدخول مجلد المشروع.

⚠️ قد تظهر رسالة تحذير حول "APP_KEY" - هذا طبيعي، سنحلها في الخطوة التالية.

ماذا تتحقق منه؟

  • تأكد من وجود مجلد flowpilot في المكان الذي نفذت فيه الأمر.
  • تأكد من وجود ملف artisan داخل المجلد (هذا هو واجهة أوامر Laravel).

02 تثبيت Breeze مع React

الهدف:

إضافة Breeze (قالب البداية) مع خيار React لتثبيت Inertia والمكتبات تلقائياً.

الخطوات:

  1. 1. تأكد أنك داخل مجلد flowpilot (استخدم cd flowpilot إذا لزم الأمر).
  2. 2. اكتب: composer require laravel/breeze --dev
  3. 3. اكتب: php artisan breeze:install react
  4. 4. اكتب: npm install (تحميل مكتبات JavaScript)

ماذا تتحقق منه؟

  • ظهرت رسالة نجاح من Breeze: "Breeze scaffolding installed successfully."
  • ملف package.json أصبح يحتوي على @inertiajs/react و react.
  • ظهر مجلد resources/js/Pages/ يحتوي على ملفات Welcome.jsx و Dashboard.jsx.

03 تشغيل الخادمين ورؤية الصفحة

الهدف:

تشغيل Laravel و Vite معاً والتحقق من ظهور الصفحة في المتصفح.

الخطوات:

  1. 1. افتح نافذة Terminal جديدة (احتفظ بالأولى).
  2. 2. في النافذة الأولى: php artisan serve
  3. 3. في النافذة الثانية: npm run dev
  4. 4. افتح المتصفح واذهب إلى: http://localhost:8000
  5. 5. التقط لقطة شاشة للصفحة التي تظهر.

💡 نصيحة: إذا لم تعمل الصفحة، تأكد من تشغيل كلا الخادمين في نفس الوقت. Vite يجب أن يعمل في نافذة Laravel في أخرى.

ماذا تتحقق منه؟

  • صفحة Laravel الترحيبية تظهر في المتصفح.
  • لا توجد أخطاء حمراء في أي من نافذتي Terminal.
  • الصفحة تستجيب (يمكنك النقر على الروابط).

★ التحدي الإضافي (اختياري)

للمتقدمين - ستندهش مما يمكنك فعله:

  1. افتح ملف resources/js/Pages/Welcome.jsx في VS Code.
  2. غيّر النص الموجود إلى "مرحباً بك في FlowPilot Academy!"
  3. احفظ الملف (Ctrl+S) - ستلاحظ أن المتصفح يتحدّث تلقائياً!
  4. هذا هو "Hot Module Replacement" (HMR) - تغيير الكود يظهر فوراً دون إعادة تحميل الصفحة.

التقط صورة للصفحة بعد التعديل - سترى النص الجديد مباشرة!

طريقة التسليم

  • • لقطة شاشة 1: Terminal بعد إنشاء المشروع (يظهر مجلد flowpilot).
  • • لقطة شاشة 2: المتصفح يعرض صفحة Laravel الترحيبية على localhost:8000.
  • • لقطة شاشة 3 (اختياري): بعد تعديل Welcome.jsx.
  • • النص: إجابات الأسئلة النظرية.
  • ⚠️ احتفظ بمجلد flowpilot - سنستخدمه في كل الدروس القادمة!

مخاطر شائعة

  • • نسيان تشغيل npm run dev: الصفحة ستظهر لكن بدون CSS وتفاعل. تأكد من تشغيل Vite.
  • • إغلاق Terminal عن طريق الخطأ: الخادم سيتوقف. أعد تشغيله.
  • • مشكلة المنافذ (Port): إذا كان port 8000 مشغولاً، استخدم php artisan serve --port=8080.
  • • عدم تثبيت Breeze أولاً: إذا حاولت تشغيل npm run dev قبل Breeze، ستفشل لأن package.json لا يحتوي React بعد.
