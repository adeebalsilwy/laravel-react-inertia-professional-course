# واجب الدرس 2: تثبيت Laravel 12 وربطه بـ React و Inertia

## الهدف
إنشاء أول مشروع Laravel حقيقي باسم FlowPilot، تثبيت Breeze مع React stack، تشغيل الخادمين، ورؤية أول صفحة في المتصفح.

## المطلوب بالتفصيل

### الجزء 1: إنشاء المشروع (إجباري)
1. افتح Terminal في المكان الذي تريد إنشاء المشروع فيه.
2. نفذ: `composer create-project laravel/laravel flowpilot`
3. انتظر حتى يكتمل التحميل (قد يستغرق 1-3 دقائق حسب سرعة الإنترنت).
4. ادخل للمجلد: `cd flowpilot`
5. تأكد من وجود الملفات التالية:
   - `artisan` (ملف واجهة الأوامر)
   - `.env` (ملف إعدادات البيئة)
   - `composer.json` (قائمة مكتبات PHP)

### الجزء 2: تثبيت Breeze مع React (إجباري)
1. داخل مجلد `flowpilot`:
   - `composer require laravel/breeze --dev`
   - `php artisan breeze:install react`
   - `npm install`
2. تأكد من ظهور مجلد `resources/js/Pages/` يحتوي على:
   - `Welcome.jsx`
   - `Dashboard.jsx`

### الجزء 3: تشغيل الخادمين (إجباري)
1. نافذة 1: `php artisan serve` (تظهر رسالة `Server running on http://localhost:8000`)
2. نافذة 2: `npm run dev` (تظهر رسالة `VITE v6.x ready`)
3. افتح http://localhost:8000 في المتصفح.
4. التقط لقطة شاشة للصفحة التي تظهر.

### الجزء 4: تحدي إضافي (اختياري)
1. افتح `resources/js/Pages/Welcome.jsx` في VS Code.
2. ابحث عن النص الترحيبي وغيّره إلى "FlowPilot - إدارة المشاريع بذكاء".
3. احفظ الملف وشاهد المتصفح يتحدّث تلقائياً.
4. التقط صورة للصفحة الجديدة.

### الجزء 5: أسئلة فهم (إجباري)
أجب بجملة أو جملتين:
- ما الفرق بين Laravel و React؟
- لماذا نحتاج إلى Inertia.js؟
- لماذا نحتاج نافذتي Terminal؟

## معيار القبول
- ظهور الصفحة الترحيبية في المتصفح دون أخطاء.
- وجود مجلد `flowpilot` كامل مع جميع الملفات.
- إجابات نظرية صحيحة.

## طريقة التسليم
- صورتان: الأولى للصفحة الترحيبية، الثانية (اختياري) للصفحة المعدلة.
- النص: إجابات الأسئلة النظرية.

## مخاطر شائعة
- لا تنس تشغيل `npm install` بعد Breeze (يتجاهلها بعض المبتدئين).
- لا تخلط بين نافذتي Terminal.
- إذا ظهرت مشكلة في port 8000، استخدم port آخر: `php artisan serve --port=8080`.
- احتفظ بالمشروع - هو قلب الدورة!
