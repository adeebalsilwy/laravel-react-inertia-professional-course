اختبر فهمك: Laravel + React + Inertia

أجب عن الأسئلة ثم اضغط "تصحيح الإجابات".

1. ما هو الدور الأساسي لـ Inertia.js في تطبيق FlowPilot؟

الشرح: Inertia.js يربط Laravel بـ React دون الحاجة لبناء APIs منفصلة. عندما يطلب المستخدم صفحة، Laravel يجهز البيانات، Inertia يمررها، و React يعرضها - كل هذا بسلاسة.

2. أي أمر تستخدم لإنشاء مشروع Laravel جديد باسم "flowpilot"؟

الشرح: Composer هو مدير حزم PHP، و create-project هو أمره لإنشاء مشروع جديد من حزمة موجودة. laravel/laravel هو اسم الحزمة في Packagist.

3. ماذا يفعل الأمر php artisan serve؟

الشرح: php artisan serve يشغل خادم PHP المدمج. هذا يسمح لك برؤية تطبيقك في المتصفح محلياً (على جهازك فقط). للرفع للإنترنت هناك طرق أخرى (مثل Laravel Forge).

4. ما وظيفة ملف routes/web.php في Laravel؟

الشرح: ملف routes/web.php يحتوي على جميع مسارات (URLs) تطبيقك. مثلاً، Route::get('/') يحدد ماذا يحدث عندما يزور المستخدم الصفحة الرئيسية. Laravel يقرأ هذا الملف لتوجيه الطلبات.

5. لماذا نحتاج إلى تشغيل npm run dev بجانب php artisan serve؟

الشرح: Vite هي أداة بناء JavaScript حديثة. تقوم بتحويل كود React (JSX) إلى JavaScript عادي يفهمه المتصفح. كما أنها تراقب تغييرات الكود وتحدث المتصفح تلقائياً (Hot Reload).

6. صح أم خطأ: Breeze هو قالب جاهز يضيف نظام مصادقة (تسجيل دخول) و Inertia مع React تلقائياً.

الشرح: صح. Laravel Breeze هو "Starter Kit" (قالب بداية). بأمر واحد php artisan breeze:install react، يضيف Inertia، React، Tailwind، ونظام تسجيل دخول كامل.

7. صح أم خطأ: يمكن العمل على FlowPilot بنافذة Terminal واحدة فقط.

الشرح: خطأ. نحتاج نافذتين: الأولى لتشغيل Laravel (php artisan serve) والثانية لتشغيل Vite (npm run dev). يمكنك فتح نافذتين في نفس Terminal أو استخدام علامتي تبويب (Tabs).

8. سؤال نظري: اشرح بأسلوبك الخاص العلاقة بين Laravel و Inertia و React. استخدم تشبيه المطعم أو أي تشبيه آخر.

مثال للإجابة: Laravel هو المطبخ (يجهز البيانات والوجبات)، React هو صالة الطعام (المكان الذي يراه الزبون)، Inertia هو النادل (يأخذ الطلب للمطبخ ويعود بالوجبة للزبون). بدون النادل، الزبون سيضطر للذهاب للمطبخ بنفسه.

9. سؤال تطبيقي: لنفترض أنك في Terminal داخل مجلد flowpilot. ما الأمر الذي تكتبه لتشغيل خادم التطوير المحلي؟

الإجابة الصحيحة: php artisan serve. هذا الأمر يشغل خادم Laravel على port 8000. إذا أردت port آخر: php artisan serve --port=8080.
