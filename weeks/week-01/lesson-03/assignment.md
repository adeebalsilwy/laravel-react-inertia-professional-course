# واجب الدرس 3: تصميم الواجهة الأولى باستخدام Tailwind CSS

## الهدف
تطبيق Tailwind CSS لتصميم واجهة ترحيبية جذابة لمشروع FlowPilot، مع تخصيص الألوان واستخدام Flexbox/Grid.

## المطلوب بالتفصيل

### الجزء 1: تخصيص الألوان (إجباري)
1. افتح `tailwind.config.js` في جذر مشروع flowpilot.
2. أضف لونين مخصصين ضمن `theme.extend.colors`:
   ```js
   colors: {
     'flowpilot': { 500: '#6366f1', 700: '#4338ca' },
     'flowpilot-accent': { 500: '#10b981', 700: '#059669' },
   }
   ```
3. احفظ الملف وأعد تشغيل `npm run dev`.

### الجزء 2: تصميم بطاقة الترحيب (إجباري)
1. افتح `resources/js/Pages/Welcome.jsx`.
2. استبدل المحتوى بالكود المذكور في الدرس (بطاقة مع عنوان، وصف، زر).
3. تأكد من استخدام الألوان المخصصة (text-flowpilot-600، bg-flowpilot-600).
4. أضف 3 بطاقات إحصائية باستخدام Grid.

### الجزء 3: تحدي إضافي (اختياري)
- أضف تأثير `animate-pulse` للشعار.
- جرب خلفية متدرجة (gradient) بألوان مبهجة.

### الجزء 4: أسئلة فهم (إجباري)
1. الفرق بين p-4 و m-4؟
2. ماذا يعني md:grid-cols-2؟
3. لماذا نستخدم className بدلاً من class في JSX؟
4. ما فائدة hover:bg-*؟

## معيار القبول
- الصفحة تظهر بتصميم جديد وجذاب.
- الألوان المخصصة (flowpilot) تعمل.
- البطاقات الإحصائية تظهر في Grid.
- خادم Vite يعمل دون أخطاء.

## طريقة التسليم
- صورتان للمتصفح (البطاقة الرئيسية + البطاقات الإحصائية).
- إجابات الأسئلة النظرية.

## مخاطر شائعة
- تأكد من تشغيل npm run_dev بعد تعديل tailwind.config.js.
- إذا لم تظهر الألوان الجديدة، تحقق من عدم وجود أخطاء في ملف الإعداد.
- لا تنس استخدام className بدلاً من class في JSX.
