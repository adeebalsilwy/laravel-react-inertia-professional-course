واجب الدرس 18: بناء لوحة مهام Kanban

"سنبني لوحة بصرية تسمح للمستخدم برؤية كل مهام مشروعه وفهم أين يقف العمل في لمحة."

الهدف من الواجب

إنشاء Task Model مع جدول tasks في قاعدة البيانات، وبناء TaskController مع دالتين (index و updateStatus)، وتصميم واجهة Kanban بثلاثة أعمدة في React. سيتعلم الطالب كيف يربط الحالة (status) بحركة المهام بين الأعمدة.

خطوات الواجب (9 خطوات)

الخطوة 1: إنشاء Task Model مع Migration

php artisan make:model Task -m

تأكد من ظهور ملفين: app/Models/Task.php وملف migration جديد.

الخطوة 2: كتابة أعمدة جدول tasks في Migration

Schema::create('tasks', function (Blueprint $table) {
  $table->id();
  $table->foreignId('project_id')->constrained()->cascadeOnDelete();
  $table->string('title', 255);
  $table->text('description')->nullable();
  $table->string('status', 20)->default('pending');
  $table->foreignId('user_id')->constrained()->cascadeOnDelete();
  $table->timestamps();
});

ثم شغّل php artisan migrate.

الخطوة 3: إضافة العلاقات في Model

في Task.php:

protected $fillable = ['project_id', 'title', 'description', 'status', 'user_id'];

public function project() { return $this->belongsTo(Project::class); }

في Project.php:

public function tasks() { return $this->hasMany(Task::class); }

الخطوة 4: إنشاء TaskController

php artisan make:controller TaskController

الخطوة 5: كتابة index()

public function index()
{
  $tasks = Task::with('project')->get();
  return Inertia::render('Tasks/Board', [
    'tasks' => $tasks,
    'success' => session('success'),
  ]);
}

تأكد من use App\Models\Task; و use Inertia\Inertia;.

الخطوة 6: كتابة updateStatus()

public function updateStatus(Request $request, Task $task)
{
  $validated = $request->validate([
    'status' => 'required|in:pending,in_progress,completed',
  ]);
  $task->update($validated);
  return redirect()->back()->with('success', 'تم تحديث الحالة');
}

الخطوة 7: تسجيل المسارات

Route::middleware(['auth'])->group(function () {
  Route::get('/tasks/board', [TaskController::class, 'index'])->name('tasks.board');
  Route::patch('/tasks/{task}/status', [TaskController::class, 'updateStatus'])->name('tasks.status');
});

شغّل php artisan route:list لتأكيد المسارات.

الخطوة 8: بناء واجهة Kanban في React

أنشئ resources/js/Pages/Tasks/Board.jsx:

import { usePage, router } from '@inertiajs/react';

export default function Board() {
  const { tasks, success } = usePage().props;
  // صنف المهام حسب status
  const pending = tasks.filter(t => t.status === 'pending');
  const inProgress = tasks.filter(t => t.status === 'in_progress');
  const completed = tasks.filter(t => t.status === 'completed');
  // اعرض 3 أعمدة مع أزرار التغيير
}

الخطوة 9: الاختبار النهائي

  • ✅ أدخل عدة مهام بحالات مختلفة (يدوياً عبر tinker أو منشئ بيانات).
  • ✅ شغّل php artisan serve واذهب إلى /tasks/board.
  • ✅ كل مهمة تظهر في العمود الصحيح.
  • ✅ اضغط زر "نقل" لتغيير الحالة → المهمة تنتقل للعمود الآخر.
  • ✅ رسالة النجاح تظهر بعد كل تغيير.
