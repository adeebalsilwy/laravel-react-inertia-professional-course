# ملاحظات الدرس 17: إنشاء المشاريع وإدارتها CRUD

## المفهوم الأساسي
- CRUD اختصار لأربع عمليات: Create (إنشاء)، Read (قراءة)، Update (تحديث)، Delete (حذف).
- كل تطبيق ويب حقيقي يستخدم CRUD لإدارة بياناته.
- في FlowPilot: المشاريع هي الوحدة الأساسية، كل مستخدم ينشئ مشاريعه ويديرها.

## أين يظهر في FlowPilot
- صفحة `/projects`: تعرض قائمة بكل المشاريع (عملية Read).
- زر "مشروع جديد": يفتح نموذجاً لإدخال اسم ووصف المشروع (عملية Create).
- زر تعديل بجانب كل مشروع: يسمح بتغيير البيانات (عملية Update).
- زر حذف بجانب كل مشروع: يحذف المشروع من قاعدة البيانات (عملية Delete).

## التحقق من البيانات (Validation)
- نستخدم `$request->validate([...])` للتأكد من صحة البيانات قبل الحفظ.
- قواعد شائعة: `required` (الحقل مطلوب)، `min:3` (الحد الأدنى 3 أحرف).
- إذا فشل التحقق، يرسل Laravel أخطاء تلقائياً إلى Inertia.

## الكود الرئيسي
- Controller: `app/Http/Controllers/ProjectController.php` يحتوي على 7 دوال.
- Route: `Route::resource('projects', ProjectController::class)` يسجل 7 مسارات تلقائياً.
- Model: `App\Models\Project` يمثل جدول المشاريع في قاعدة البيانات.

## تدفق البيانات
1. المستخدم يضغط زر في React → 2. Inertia يرسل طلب HTTP → 3. Laravel يستقبل في Controller → 4. يتفاعل مع قاعدة البيانات → 5. يعيد النتيجة إلى React.

## أخطاء شائعة
- نسيان `Route::resource` يؤدي إلى صفحة 404.
- عدم تمرير `'projects' => Project::all()` إلى Inertia يؤدي إلى صفحة فارغة.
- نسيان `use App\Models\Project;` يسبب خطأ Class not found.
- عدم استخدام `@csrf` أو `_method: 'put'` في النموذج يسبب خطأ 419.

## نصائح
- ابدأ باختبار دالة `index()` ثم أضف `create()` ثم `store()` وهكذا.
- استخدم `php artisan route:list` لرؤية كل المسارات المسجلة.
- تأكد من وجود `fillable` في Model للسماح بالتعبئة الجماعية.
