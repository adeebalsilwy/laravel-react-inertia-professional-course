كويز: إنشاء المشاريع وإدارتها CRUD

أجب عن جميع الأسئلة، ثم اضغط "تصحيح" لترى النتيجة.

السؤال 1 (اختيار من متعدد): ماذا تعني أحرف CRUD؟

✅ الشرح: CRUD اختصار لـ Create (إنشاء)، Read (قراءة)، Update (تحديث)، Delete (حذف). هذه العمليات الأربع هي أساس أي تطبيق يدير بيانات.

السؤال 2 (اختيار من متعدد): كم عدد المسارات التي يسجلها Route::resource؟

✅ الشرح: Route::resource ينتج 7 مسارات: index, create, store, show, edit, update, destroy. كل مسار يقابل دالة في Controller.

السؤال 3 (اختيار من متعدد): ما الـ Middleware الذي نستخدمه لحماية مسارات CRUD؟

✅ الشرح: Middleware 'auth' يمنع المستخدمين غير المسجلين من الوصول إلى المسارات. نضيفه كالتالي: Route::middleware(['auth'])->group(function () { Route::resource('projects', ProjectController::class); });

السؤال 4 (اختيار من متعدد): ما الخاصية التي نضيفها في Model لمنع خطأ MassAssignmentException؟

✅ الشرح: $fillable تحدد الحقول التي يسمح بتعبئتها بشكل جماعي (Mass Assignment). مثلاً: protected $fillable = ['name', 'description', 'user_id'];

السؤال 5 (اختيار من متعدد): ما الدالة التي تجلب كل السجلات من جدول معين في Eloquent؟

✅ الشرح: Model::all() ترجع كل السجلات في الجدول كمصفوفة (Collection). مثال: $projects = Project::all();

السؤال 6 (صح/خطأ): يمكن استخدام Inertia::render() بدون تمرير بيانات (props).

✅ الشرح: صح. المعامل الثاني في Inertia::render() اختياري. مثلاً: Inertia::render('Projects/Create') لا يحتاج props لأنه مجرد نموذج فارغ.

السؤال 7 (صح/خطأ): دالة destroy() تحتاج إلى استقبال كائن Project عبر Route Model Binding.

✅ الشرح: صح. Laravel يستخدم Route Model Binding لجلب كائن Project تلقائياً بناءً على id في الرابط. توقيع الدالة: public function destroy(Project $project)

السؤال 8 (نظري): اشرح بجملتين أو ثلاث كيف يتعامل Laravel مع طلب POST من Inertia لإنشاء مشروع جديد.

✅ الإجابة النموذجية: يرسل Inertia طلب POST إلى /projects مع بيانات النموذج و CSRF token. يستقبل ProjectController دالة store() التي تتحقق من صحة البيانات عبر validate(). إذا نجح التحقق، ينشئ Laravel سجلاً جديداً في جدول projects باستخدام Project::create(). ثم يعيد التوجيه إلى صفحة القائمة مع رسالة نجاح (flash message) عبر with().

السؤال 9 (عملي): اكتب دالة store() كاملة تتضمن التحقق من أن الاسم مطلوب وأكبر من 3 أحرف، مع رسالة نجاح.

✅ الإجابة النموذجية:
public function store(Request $request)
{
  $validated = $request->validate([
    'name' => 'required|string|min:3|max:255',
    'description' => 'nullable|string|max:1000',
  ]);
  $validated['user_id'] = auth()->id();
  Project::create($validated);
  return redirect()->route('projects.index')->with('success', 'تم إنشاء المشروع بنجاح');
}
