التعامل مع النماذج (Forms): قوة أداة useForm في Inertia

"لا تجعل المستخدم ينتظر في صمت! لنتعلم كيف نبني نماذج ذكية تخبر المستخدم بما يحدث خلف الكواليس."

لماذا نستخدم useForm؟

في تطبيقات React العادية، ستحتاج لكتابة كود طويل لإدارة حالة المدخلات (Inputs) والأخطاء وحالة الإرسال. Inertia يسهل عليك المهمة عبر useForm.

توفر لك هذه الأداة كل ما تحتاجه في سطر واحد: البيانات، الأخطاء، وهل يتم الإرسال حالياً أم لا؟

حالات التحميل (Processing States)

من أسوأ تجارب المستخدم هي الضغط على زر "حفظ" وعدم حدوث شيء. يجب أن يرى المستخدم كلمة "جاري الحفظ..." أو يرى الزر أصبح باهتاً (Disabled) ليشعر بأن التطبيق يعمل.

استخدام الـ Form Hook

const { data, setData, post, processing, errors } = useForm({

  name: '',

  description: ''

});

ثم استخدم processing لتعطيل الزر:

<button disabled={processing}>

  {processing ? 'جاري الإرسال...' : 'حفظ المشروع'}

</button>
